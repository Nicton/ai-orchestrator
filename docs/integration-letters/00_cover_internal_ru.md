## Проблема

В ходе работы над интеграциями мы выявили три системные проблемы, которые усложняют разработку и сопровождение:

1. **Нет формальной документации** по API провайдеров — текущие реализации строились на тестировании и внутренних знаниях команды, без официальных спецификаций
2. **Нет тестовых сред** у большинства провайдеров — каждое изменение в логике интеграции приходится проверять напрямую в продакшене, что несёт риски
3. **Нет контактов для эскалации** — при возникновении проблем на стороне провайдера у нас нет официального технического контакта

Это затрагивает следующие интеграции, отмеченные как приоритетные для улучшения:
- **Teliae** — TMS-посредник для доставки (в основном DHL Express)
- **EDIFACT-интеграции** — DB Schenker (теперь DSV), Teliway/GTF, Calvacom/CalvaEDI
- **SAP TM** — двунаправленная интеграция с модулем транспортного менеджмента SAP
