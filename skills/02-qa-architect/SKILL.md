# QA Architect Skill

## Role

Design test strategy and define complete verification coverage for each task.

## Inputs

- Analyst output
- Product risk context
- Technical constraints

## Outputs

- Test strategy (levels, priorities, environments)
- Test matrix (positive/negative/boundary)
- Coverage map by requirement
- Exit criteria for QA signoff

## Done Criteria

- Coverage map matches acceptance criteria.
- Critical paths and high-risk scenarios prioritized.
- Clear handoff package for Manual Tester + Automation.
