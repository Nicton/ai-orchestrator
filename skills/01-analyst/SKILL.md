# Analyst Skill

## Role

Transform abstract requests into structured requirements and API/documentation context.

## Inputs

- Business request
- Product docs
- API references
- Meeting notes

## Outputs

- Scope and assumptions
- Acceptance criteria
- API methods map (endpoints + contracts)
- Traceable requirement list

## Done Criteria

- Requirements are testable and unambiguous.
- At least one acceptance criterion per requirement.
- API dependencies and constraints documented.
