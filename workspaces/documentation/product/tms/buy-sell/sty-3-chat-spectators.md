# STY 3.0 — Чат, навигация BUY/SALES и Spectators

> Источник требований: REQ-STY-004..008 | Слайды: 2023 02 - STY 3.0

---

## Дублирование чатов при создании SH из QR (REQ-STY-004)

В STY 3.0 цепочка объектов создаёт **связанные чаты** для полной прозрачности коммуникации.

### Правила создания чатов

| Событие | Что создаётся |
|---------|--------------|
| Создание SH из QR | Новый чат на уровне SH с **копией истории чата QR** |
| Создание SH из QR | Новый чат на уровне SH с **копией истории чата BK** |
| Сообщение на уровне подтверждённого QR | Копируется во **все связанные SH** |
| Сообщение на уровне PSH | Копируется в связанный SH и QR |

### Привязка чатов

Чат может быть привязан к следующим объектам:
- **PSH** (Pre-Shipment / заявка)
- **QR** (Quote Request / запрос котировки)
- **BK** (Booking)
- **SH** (Shipment)

При переходе от QR → SH история переписки сохраняется на обоих уровнях.

## Навигация BUY / SALES (REQ-STY-005)

TBS-аккаунт видит две стороны платформы с отдельными навигационными структурами.

### SALES side (Can Sell = YES)

| Раздел | Описание |
|--------|---------|
| Booking | Входящие запросы от 3PL Customers |
| Grouping | Группировка заявок |
| Tracking | Трекинг исходящих отправок |
| Invoicing | Счета и расчёты с покупателями |
| Claims | Рекламации |
| Multivision | Мультимодальный обзор |
| Dashboard | Дашборд продаж |

**Правила:**
- Переключатель **BUY/SALES** доступен авторизованным пользователям (Can Sell = YES)
- BUY side — видит бронирования "to" нужным аккаунтам (классический TMS)
- SALES side — видит бронирования "from" (запросы от клиентов)
- Новый раздел **"Requests"** (EN) / **"Demandes"** (FR) создаётся отдельно от "Booking"
- Французское название "Booking" переименовывается из "Demandes" в "Booking"

## Общие настройки MD и документов по умолчанию (REQ-STY-006)

Настройки по умолчанию задаются на уровне метаданных (MD) и документов:

| Уровень применения | Когда срабатывает |
|-------------------|-----------------|
| Создание BK (Booking) | Настройки MD и Doc применяются при создании бронирования |
| Создание SH (Shipment) | Настройки применяются при создании отправки |

### Чат-подписчики

- Подписчики чата **BK** формируются по персональным настройкам пользователей при создании BK
- Подписчики чата **SH** формируются по персональным настройкам пользователей при создании SH
- Изменение настроек **не затрагивает** уже существующие чаты

## Spectators в контексте BK и QR (REQ-STY-007)

Spectators (наблюдатели) получают расширенный доступ к объектам Buy&Sell.

### Уровни доступа Spectator

| Объект | Доступ |
|--------|--------|
| **BK (Booking)** | Spectators добавляются на уровне BK |
| **QR (Quote Request)** | Доступ ко всем **публичным группам чата** каждого QR |
| **QR — информация** | **Ограниченный**: только средняя колонка, без ставок |

### Группы чата

| Группа | Видимость |
|--------|----------|
| **Public** | Все участники сделки |
| **Private** | Только моя команда (аккаунт) |
| **Private Carrier/Shipper** | Только перевозчик/шиппер |

**Правило:** любой пользователь аккаунта может иметь доступ к любой группе чата, где аккаунт активирован.

## Управление уведомлениями пользователя (REQ-STY-008)

Отдельная вкладка **"Notifications"** в профиле пользователя заменяет прежние inline-настройки.

### Страница уведомлений

- Секция уведомлений **удалена** из вкладки "General" — перенесена в отдельную вкладку
- Кнопка **"Notify"** заменяет прежнюю логику на `SEND ONLY CHAT MESSAGE NOTIFICATIONS`

### Carrier: настройка по клиентам

На carrier-стороне доступен раздел **"Customers"** для фильтрации уведомлений:
- Пользователь может активировать / деактивировать уведомления по конкретным клиентам (Galaxies)
- Позволяет избежать нотификационного шума от нерелевантных клиентов
