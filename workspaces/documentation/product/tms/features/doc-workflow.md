# Documents Workflow — Уведомления о документах

> Источник требований: REQ-DOC-001..011 | Слайды: 2024 11 - Documents workflow, 2023 02 - STY 3.0

---

## Концепция

Documents Workflow — система персонализированных уведомлений о событиях с документами (загрузка, удаление, изменение видимости). Каждый пользователь настраивает свои workflow-правила в профиле.

**Настройка доступна только через профиль пользователя → вкладка Notifications**, а не через общие настройки аккаунта.

---

## Создание workflow уведомления (REQ-DOC-001)

Пользователь может создать один или несколько workflow-правил. Каждое правило описывает: **какой тип документа** + **в каких условиях** + **при каком действии** → уведомить через **какой канал**.

При дублировании условий — пользователь **не получает двойное оповещение**.

Workflow работает как для API-потоков, так и для ручных загрузок.

---

## Типы документов (REQ-DOC-002)

- Список типов документов ограничен теми, что включены на уровне аккаунта
- Пользователь добавляет один или несколько типов в один workflow
- Без выбора хотя бы одного типа — workflow не сохраняется

Примеры поддерживаемых типов: Proof of Delivery, Bill of Lading, Commercial Invoice.

---

## Условия триггера (REQ-DOC-003)

| Условие | Описание |
|---------|----------|
| Я владелец объекта | Booking, Shipment, TR, Slots |
| Все потоки через API на этом аккаунте | Автоматические интеграции |
| Все потоки созданные вручную | Ручные загрузки |
| Все потоки на аккаунте | Дефолт — охватывает всё |

При multi-account: настройки применяются к каждому аккаунту отдельно.

---

## Действия-триггеры (REQ-DOC-004)

| Действие | По умолчанию |
|----------|-------------|
| **Added** — документ загружен | Включено |
| **Removed** — документ удалён или отменён | Настраивается |
| **Visibility changed** — изменена видимость документа | Настраивается |

Должно быть выбрано минимум одно действие.

---

## Географический периметр (REQ-DOC-005)

Фильтр **FROM / TO** по Country или LogZone — опциональный. При отсутствии применяется ко всем потокам.

Пример: уведомлять только о POD из `FR95` → Испания.

Периметр отображается в строке сводки workflow.

---

## Каналы доставки (REQ-DOC-006)

| Канал | Примечание |
|-------|-----------|
| **Notification** (in-app) | Выбран по умолчанию |
| **Mail** | — |
| **Notification & Mail** | Оба канала |

Настройка канала независима для каждого workflow.

Multi-account пользователи получают уведомления для всех затронутых аккаунтов.

---

## Список workflow и управление (REQ-DOC-007..008)

Каждый workflow отображается строкой: документы | счётчик аккаунтов | условие | действия | периметр | канал.

- Клик на строку → просмотр и редактирование
- Кнопка «Add a workflow» — создать новое правило
- Иконки аккаунтов в строке для визуальной идентификации

**Выбор аккаунтов:**
- По умолчанию предвыбирается основной аккаунт пользователя
- Добавить другие аккаунты из MULTI ACC setup (BO + CONSTELLATION)
- Если пользователь привязан только к одному аккаунту — список аккаунтов скрыт

---

## Уровни доступа к документам в группах (REQ-DOC-009)

| Уровень | Кто видит документ |
|---------|--------------------|
| **Private** | Только мой аккаунт |
| **Limited** | Только Shipper и Carrier |
| **Public** | Все аккаунты с доступом к отправке; новые участники, добавленные позже, получают доступ автоматически |
| **Specific** | Выбранные вручную аккаунты |

---

## Передача документов по типу группировки (REQ-DOC-010)

| Тип группы | Варианты распространения |
|-----------|--------------------------|
| Multi Container | This shipment / All shipments / Selection / **Same Ship** |
| Grouped Departure | This shipment / All shipments / Selection / **Same Destination** |
| Grouped Delivery | This shipment / All shipments / Selection / **Same Pick-up** |
| Grouped Shipments | This shipment / All shipments / Selection |
| Milkrun | This shipment / All shipments / Selection |

---

## Уведомления при загрузке документа в группе (REQ-DOC-011)

- Уведомления в чате отправляются **только** аккаунтам, которым документ был предоставлен
- Вместо полного списка — числовой тег (например, `12`)
- Mouseover на тег → список предоставленных аккаунтов и уведомлённых пользователей
- Тег показывает уровень: `private` / `limited` / `specific` (public — без тега)
- Уведомление не отправляется повторно, если пользователь уже был уведомлён вручную
