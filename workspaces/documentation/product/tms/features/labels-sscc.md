# Labels & SSCC — Штрихкоды и мульти-лейблы

> Источник требований: REQ-LABEL-001..006 | Слайды: 2025 06 - Barcode generation and Multi Label triggering

---

## Концепция

SSCC (Serial Shipping Container Code) — уникальный штрихкод для каждой единицы груза. Shiptify генерирует SSCC автоматически при переходе PSH → SH, если функция активирована на аккаунте.

Вдохновлено стандартом GS1-128, но **не требует регистрации в GS1**.

---

## Активация на уровне аккаунта (REQ-LABEL-001)

В настройках покупателя (buyer) → опция **«Enable SSCC on labels»** (по умолчанию: OFF).

При включении:
- **PREFIX** становится обязательным полем (ровно 7 цифр, уникален в базе данных Shiptify)
- Метки генерируются автоматически на уровне единицы, а не по запросу
- Метки сохраняются в менеджере документов

---

## Структура SSCC кода (REQ-LABEL-002)

```
Extension digit (1) + Company prefix (7) + Serial reference (9) + Check digit (1) = 18 символов
```

| Часть | Длина | Управление |
|-------|-------|-----------|
| Extension digit | 1 | Фиксированный |
| Company prefix | 7 | Управляется вручную на уровне аккаунта, уникален в базе |
| Serial reference | 9 | Последовательный счётчик на уровне аккаунта |
| Check digit | 1 | Рассчитывается по алгоритму modulo 10 от первых 17 цифр |

---

## Trigger генерации (REQ-LABEL-003)

SSCC генерируется если одновременно выполнены два условия:

1. SSCC включён на уровне аккаунта
2. Для конкретного перевозчика включён флаг **«Generate barcodes»** (по умолчанию: NO)

**Момент генерации:** при переходе PSH → SH (VALIDATING BOOKED SR/PSH).

Сгенерированный SSCC включается в webhook «create shipment» для перевозчика.

Режим генерации: **мульти-лейбл** вместо единственной метки.

---

## Хранение SSCC в структуре данных (REQ-LABEL-004)

- SSCC хранится в поле **FREIGHT UNIT KEY** на уровне контента
- При `Quantity > 1` и активном trigger → контент **автоматически разбивается** на строки по 1 единице
- Каждая строка с `Qty=1` получает уникальный SSCC
- Webhook возвращает `freight_unit_key` для каждой строки контента
- SSCC доступен через API, UI и Elastic Search

---

## Управление изменениями пакинг-листа при SSCC (REQ-LABEL-005)

| Действие | Поведение с SSCC |
|----------|-----------------|
| Обновление пакинг-листа | Метки регенерируются, SSCC **не** перегенерируются |
| Добавление новой единицы | Новый SSCC для новой строки; существующие не меняются |
| Удаление единицы | Удаляется только SSCC удалённой строки |

Все метки хранятся в одном PDF (аналогично текущему скачиванию мульти-лейблов).

---

## Совместимость с workflow отправки (REQ-LABEL-006)

- Мульти-лейблы с SSCC совместимы с workflow **«Send by Mail»** без изменений
- Клик «Generate N labels» → скачивает лист меток **без сохранения** в Shiptify
- Одиночная метка (Single label) пересоздаётся в менеджере документов
- Будущее: поддержка формата **ZPL 10x10**
