# Visual Indicators — Визуальные индикаторы слотов

> Обновлено 2026-06-12 по комментариям Product: добавлен бизнес-контекст, исправлен термин PML.

## Зачем это создано (бизнес-контекст)

**Проблема:** перевозка может требовать бронирования слота на складе — а может не требовать. Если перевозчик об этом не знает, машина приезжает «в никуда»: разворот у ворот, простой, штраф, срыв окна доставки. Диспетчеры выясняли это звонками: «а как у вас бронироваться?»

**Решение:** цветовой индикатор прямо в списке перевозок и на Tracking-вкладке отвечает на оба вопроса за один взгляд: **нужен ли слот** и **как его получить**. Это снижает no-show на складах, убирает звонки-выяснения и делает требования площадки видимыми обеим сторонам сделки одинаково (Shipper и Carrier видят одно и то же — нет «у меня в системе по-другому»).

**Откуда берётся:** индикатор — производная от настройки локации в адресной книге (поле `slot_type` на Location). Сущность появилась вместе с PML-программой: когда склады начали управлять слотами через Slotify, потребовалось отличать «бронируйся у нас в системе» от «бронируйся по телефону» и «слот не нужен».

## Значения и что делать пользователю

| Цвет | Тип | Что это значит | Действие перевозчика |
|------|-----|----------------|----------------------|
| 🟢 Зелёный | **PML (Public Master Location)** | Локация управляет слотами через Shiptify/Slotify | Бронировать слот в системе (кнопка/ссылка Slotify); hover-popup: «Online: [название]» + инструкции |
| 🟠 Оранжевый | **Manual Slot Booking Required** | Слот обязателен, но бронируется вручную **или** через внешнюю систему площадки (портал, телефон, email) | Следовать инструкции из popup: «Manual: [как бронировать]» + контакты |
| ⚫ Нет индикатора | Обычная перевозка | Для этой локации слот-процесс **не настроен владельцем** (slot_type = None) — бронирование слота не требуется | Ехать по обычному процессу |

### «Слоты не настроены для этой локации» — что это значит и как настроить

Это состояние **slot_type = None**: владелец локации не объявил требований к слотам. Если на самом деле площадка требует слоты — это пробел настройки, а не свойство перевозки. Настройка: **Address Book → карточка локации → Slot type** (None / PML / Manual Required) + текст инструкции для перевозчика. Для значения PML локация должна быть подключена как Public Master Location (см. [PML Settings](../../dock/feature-docs/pml-settings/README.md)).

## Где отображается

- Список перевозок `/shipments` — первый столбец
- Деталь перевозки → вкладка Tracking
- Обе роли (Shipper, Carrier) — одинаково

## Сценарии использования

1. **Диспетчер перевозчика** планирует день: фильтрует рейсы, по зелёным бронирует слоты в Slotify пакетно, по оранжевым ставит задачи «позвонить на склад».
2. **Шиппер** проверяет, почему груз «завис»: оранжевый индикатор без подтверждённого слота = вероятная причина — слот не забронирован вручную.
3. **Онбординг новой локации клиента**: пока slot_type=None, перевозчики не получают требований — заполнение карточки локации сразу «включает» правило для всех будущих перевозок.
