# Expected Orders (EO) — Ожидаемые заказы

> Источник требований: REQ-EO-001..007 | Слайды: 2024 02 - EO new object + Collaborative Order

> ⚠️ **СТАТУС ПО КОДУ (2026-06-11): EO как сущность НЕ реализована.** В коде существуют только модель `Order` (статусы PENDING/ASSIGNED/DELIVERED/CANCELLED/MISSING_SLOT, одна пара buyer/seller partner на заказ), Dock Order/SDO и `SlotOrder`. Миграционных скриптов EO нет; уведомлений поставщику при создании заказа нет; статуса REJECTED нет. Этот документ — спецификация будущей фичи; вопросы ниже относятся к её дизайну.

---

## Концепция

Expected Order (EO) — новый объект, заменяющий Dock Order в интерфейсе PML (Platform Managed Logistics). EO описывает **ожидаемую поставку** от поставщика: что, куда и когда должно прийти. Слот бронируется поставщиком под конкретный EO, после чего создаётся Order.

**Иерархия объектов:**
```
EO (Expected Order)
└── ORDER (создаётся при бронировании слота)
    └── Slot
```

**Ключевое отличие от Dock Order:**
- EO управляется PML (покупателем/складом), ORDER создаётся перевозчиком/поставщиком при бронировании
- Загрузка XLS обновляет базу EO, а не базу ORDER
- WH Customer определяется на уровне ORDER, а не на уровне SLOT

## Структура объекта EO (REQ-EO-001)

| Поле | Тип | Описание |
|------|-----|----------|
| UID | string | Уникальный ID на уровне PML/Supplier ID |
| EO_Name | string | Название заказа |
| EO_Zone | string | Зона доставки |
| EO_WH_Customer_ID | string | ID клиента склада |
| EO_Supplier_ID | string | ID поставщика |
| EO_Supplier_Name | string | Имя поставщика |
| EO_Supplier_Email | string | Email поставщика |
| EO_Packing_List_Qty | number | Количество в пакинг-листе |
| EO_Packing_List_Cargo_Type | string | Тип груза |
| EO_Content_Parcels | number | Количество посылок |
| EO_Content_Units | number | Количество единиц |
| EO_Delivery_Min_date | date | Минимальная дата доставки |
| EO_Delivery_Max_date | date | Максимальная дата доставки |
| EO_Creation_Date | date | Дата создания EO |
| EO_Tag | string | Теги для категоризации |
| EO_Auto_Close | boolean | Автоматическое закрытие |

**Статусы EO:**

| Статус | Описание |
|--------|----------|
| Open | EO активен, ожидает бронирования |
| Closed | EO закрыт (вручную или автоматически) |
| Cancelled | EO отменён |
| Ongoing | Автоматический — установлен системой при активном процессе |

## Связь EO с заказами и слотами (REQ-EO-002)

- Один ORDER может быть назначен на **N слотов** (связь N:N через EO)
- При бронировании слота создаются ORDER из матрицы EO
- EO хранит все данные, которые ранее хранились в SLOT ORDER
- WH Customer определяется на уровне **ORDER**, а не на уровне SLOT

## Бронирование слота через EO (REQ-EO-003)

Процесс бронирования:

1. При бронировании слота система ищет соответствующие EO (вместо Dock Order)
2. Carrier/Supplier выбирает, что хочет доставить: паллеты, коробки, единицы
3. Логическая проверка соответствия **не выполняется** (доставщик знает лучше)
4. После бронирования создаётся ORDER, привязанный к слоту и к родительскому EO
5. Зона слота по умолчанию берётся из EO, но может быть изменена **без влияния на EO**

## Управление статусами EO (REQ-EO-004)

PML-пользователь может менять статус EO вручную:

| Действие | Доступно |
|----------|----------|
| → Open | Вручную |
| → Closed | Вручную |
| → Cancelled | Вручную |
| → Ongoing | Только система |

Клик по статусу открывает модал с полем «Comment» для подтверждения. Логи хранятся для будущей интеграции с CHAT.

---

## Список EO — интерфейс PML (REQ-EO-005)

Список EO заменяет список Dock Order в интерфейсе PML.

**Колонка слотов:**
- Серый — количество отменённых
- Голубой — ожидающих
- Зелёный — доставленных без инцидентов
- Красный — доставленных с инцидентами

**Строка EO показывает:**
- L1: Supplier Name
- L2: WH Customer Name | Zone
- Логотип: если EO_WH_Customer_ID заполнен — логотип клиента; иначе — логотип PML

**Клик на колонку слотов:** модал со списком слотов (имя, дата, статус, пакинг-лист).

---

## Представление EO в аккаунте Supplier (REQ-EO-006)

В аккаунте поставщика:

- Появляется раздел **«Orders»** в левом меню (первая позиция)
- Отображается список EO, доступных поставщику
- Строка EO: L1 — PML Name; L2 — LogZone, City, Country
- Статусы для Supplier: Open, Closed, Cancelled, Pending, Booked, Delivered
- Клик на слот → открывается в новой вкладке

## Миграция данных: Dock Order → EO (REQ-EO-007)

| Исходный статус Dock Order | Целевой статус EO |
|---------------------------|-------------------|
| PENDING | Open |
| MISSING SLOT | Open |
| ASSIGNED | Closed |
| DELIVERED | Closed |

**Правила миграции:**
- API владельца PML обновляет базу EO, а не базу ORDER
- Существующий UX загрузки XLS сохраняется, но данные пишутся в EO
- После миграции новые слоты автоматически используют EO
