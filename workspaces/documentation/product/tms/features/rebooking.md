# Repeat — Повтор перевозки

> Обновлено 2026-06-12 по комментариям Product: в продукте фича называется **Repeat** (понятия «Duplicate» нет); не путать с кнопкой **Replan** (перенос дат существующей перевозки) и с Repeat для recurring slots в DOCK.

Repeat — функция создания новой перевозки на основе существующей. Копирует все данные исходной перевозки, кроме полей, специфичных для конкретного рейса.

## Кто использует

- **Shipper** — дублирует перевозку для повторяющихся маршрутов
- Недоступно для перевозок, созданных через роль **Booker** (бронирование от имени другого аккаунта) — повторять может только владелец

## Смежные функции

| Функция | Отличие |
|---------|---------|
| **Replan** (кнопка на перевозке) | Переносит даты ТЕКУЩЕЙ перевозки (+reason code), новая не создаётся |
| **Repeat Request** (Buy&Sell, уровень TR) | Повтор запроса транспорта — см. buy-sell/repeat-request.md |
| **Recurring slots** (DOCK) | Регулярное расписание слотов по шаблону — см. dock/feature-docs/pml-settings |

## Как найти

На странице списка перевозок (`/shipments`):
- Наведите мышь на первый столбец строки (hover)
- Появляется иконка "повтор" 🔄
- Клик → открывается CSW Wizard с предзаполненными данными

## Что копируется при Rebooking

| Поле | Копируется? |
|------|-------------|
| Entities (организации) | ✅ Да |
| Tags | ✅ Да (на уровне BK) |
| Откуда / Куда (маршрут) | ✅ Да |
| Тип перевозки (LTL/FTL) | ✅ Да |
| Груз (параметры) | ✅ Да |
| Документы | ✅ Да |
| Перевозчик | ✅ Да |

## Что НЕ копируется

| Поле | Причина |
|------|---------|
| Shipment name | Уникальное для каждой перевозки |
| Pick-up date | Специфично для конкретного рейса |
| Delivery date | Специфично для конкретного рейса |
| Статус | Новая перевозка начинает с Draft/Pending |
| TrackingPoints | Создаются заново |
| Invoice | Создаётся после выполнения |

## Поток

```
1. Список перевозок → hover строки → клик на иконку 🔄
2. Открывается CSW Wizard с предзаполненными данными
3. Пользователь меняет только дату (и что нужно)
4. SEND BOOKING → новая перевозка создана
```

## Ограничения

- Booker-перевозки (созданные от имени другого аккаунта) не могут быть продублированы
- Если перевозчик был удалён из системы — поле Carrier в CSW будет пустым

## Бэкенд

- Endpoint: `POST /api/v1/shipment-requests/duplicate`
- Frontend: `workspaces/frontend/public/app/shipments/` → duplicate action
