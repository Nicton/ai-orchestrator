## Концепция

Pallets Management — модуль учёта и движения поддонов между сайтами. Позволяет:
- Вести инвентарь поддонов на каждом складском сайте
- Фиксировать движения поддонов между контрагентами
- Запрашивать и подтверждать перемещение поддонов

**Активация:** раздел «Pallets Management» в левом меню появляется только на сайтах с активированным модулем `Pallets_Mgt`.

## Инвентаризация поддонов (REQ-FU-009)

**Публичный инвентарь (Public Inventory)** управляется вручную:

| Поле | Описание |
|------|----------|
| Тип поддона | Тип (EUR, UK, Chep, ...) |
| Good qty | Количество хороших поддонов |
| Fair qty | Количество подходящих |
| Broken qty | Количество сломанных |
| Pal/Column | Количество паллет на колонну |
| Min | Минимальный запас |
| Order | Кратность заказа |

**Режимы инвентаризации:**
- На уровне колонок
- На уровне отдельных поддонов

При обновлении отображается: **разница (+/- количество)** и новое итоговое значение.

Отрицательный баланс **разрешён** (не блокируется). Пример: `-213 units`.

## Движения поддонов (REQ-FU-010)

Каждое движение имеет контрагента: внутренний аккаунт Galaxy или внешний сайт.

**Статусы движения:**

| Статус | Описание |
|--------|----------|
| Inventory | Инвентаризационная запись |
| Pending | Создано, видно обеим сторонам |
| Confirmed | Контрагент подтвердил |
| Arbitration | Контрагент не согласен |
| Cancelled | Отменено |

**Автоматическое создание движения:**
- При подтверждении погрузки (Pick Up TP) → модал для указания количества консигнационных поддонов → движение создаётся в статусе `Pending`
- При подтверждении доставки (Delivery TP) → движение переходит в `Confirmed` (или подтверждается существующее Pending-движение)

## Запрос на перемещение поддонов (REQ-FU-011)

**Manual Transfer:** пользователь инициирует перемещение к другому сайту.

**Форма запроса:**
- Email получателя
- Количество поддонов
- Качество: Fair / Good

**Логика отправки:**
- Если получатель — пользователь Shiptify → заявка создаётся в системе
- Если нет — генерируется **PDF-файл**

**Book/Request (запрос на получение поддонов):**
- Учитывает Min Order и кратность (Multiple of N)
- Отображается доступное количество у контрагента

**Accounting Management:**
- Показывает баланс по каждому партнёру: **In / Out / Balance**

**Мультиаккаунтовый дашборд** в DASHBOARDS — агрегированный обзор по всем сайтам.
