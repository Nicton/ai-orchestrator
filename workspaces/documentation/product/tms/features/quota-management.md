## Концепция

Quota Management — модуль распределения объёма перевозок между перевозчиками по заданным долям. Позволяет шипперу задать, какой процент отправок должен обрабатывать каждый перевозчик на конкретном направлении, и контролировать соблюдение этих пропорций.

**Пример использования:**
- Carrier A — 60% объёма из FR95 → Испания (Road)
- Carrier B — 30%
- Carrier C — 10%

При достижении 90% квоты системой отправляется алерт.

## Активация модуля (REQ-QUOTA-001)

**В BO:** раздел `buyers/functions` → добавить раздел **Quotas** для активации на аккаунте.

**После активации:** в admin panel аккаунта появляется раздел **Quotas** в секции Advanced General.

Квоты применяются к определённым:
- Режимам транспортировки
- Направлениям (logzone from → logzone to)

---

## Создание и настройка квоты (REQ-QUOTA-002)

При создании квоты пользователь задаёт:

| Поле | Описание |
|------|----------|
| Режим транспортировки | Один или несколько (Road, Air, Sea, etc.) |
| Направление | From LogZone → To LogZone |
| Распределение % | Доли по каждому перевозчику (сумма = 100%) |
| Типы бронирования | Применимые типы |
| База расчёта | По чему считается квота |
| Частота | Период расчёта |
| База дат | Дата отправки или дата доставки |
| Exception management | Правила для исключений |

**Управление квотами:** список существующих квот с возможностью редактирования и удаления.

## Отображение квот при создании RFQ (REQ-QUOTA-003)

В экране RFQ (Request for Quotes) перевозчики **сортируются по квоте** — от наименее использованной к наиболее:

**Для каждого перевозчика отображается:**
- Текущее использование квоты (%)
- Прогнозируемая квота
- Общий объём (tooltip с числами)

**Чекбокс «Исключить перевозчиков с превышением квоты»:**
- Фильтрует список перевозчиков
- Если все превысили квоту и чекбокс активен → состояние **«Zero Match»** с предупреждением

## Пороговые алерты и автоматический фоллбэк (REQ-QUOTA-004)

**Ранний алерт:** уведомление транспортной команде при достижении порога квоты (например, 90%).

**Опция Automated Fallback** при превышении всех квот:

| Вариант | Поведение |
|---------|-----------|
| Block shipment | Блокировать создание отправки |
| Warn user | Предупредить пользователя, но разрешить |
| Push to Spot Market | Автоматически отправить на спот-рынок |

**Edge case:** отгрузки, забронированные в конце месяца A, но доставленные в месяц B — обрабатываются согласно настройке «База дат» (Delivery Date + Monthly).

## Dashboard и отчётность по квотам (REQ-QUOTA-005)

**Dashboard Accrual View:**
- Ожидаемое потребление квоты (ongoing transits — в пути)
- Фактическое потребление (delivered — доставлено)

**KPI трансформации:**
- Среднее время ответа на QA (MIN/MAX/AVG)
- Процент awarded QA
- Процент отказов

**Видимость для перевозчика:** перевозчик видит своё потребление квоты в своём портале.

**Quality Score:** возможность добавить % OTD (On-Time Delivery) рядом с показателем квоты в списке RFQ.
