## Паттерн интеграции (единая структура)

Все интеграции следуют одному паттерну:

```
app/services/integration/[carrier]/
├── service.js       # 1. Точка входа — кладёт задачу в очередь
├── impl.js          # 2. Оркестрация — координирует весь флоу
├── dataResolver.js  # 3. Загружает данные из БД
├── dataBuilder/     # 4. Трансформирует в формат перевозчика
├── provider.js      # 5. HTTP-клиент к API перевозчика
├── constants.js     # Константы интеграции
└── tasks.js         # Cron-задачи (polling, обновления)
```

### Поток интеграции

```
service.js          → addToQueue('integrationXxx', params)
                           ↓
worker picks up job
                           ↓
impl.js             → orchestrate()
    ├── dataResolver.js    → loadShipment(), loadCarrier()...
    ├── dataBuilder/       → buildRequest() → формат перевозчика
    └── provider.js        → POST /carrier-api/book
                           ↓
Обновление БД: tracking_number, status
                           ↓
Уведомление: email/webhook
```

### Код `service.js` (точка входа)

```javascript
const addToQueue = (task, params) => new Promise((resolve, reject) => {
    queue.create('integrationDhl', { task, params })
        .save(err => err ? reject(err) : resolve());
});

module.exports = {
    bookShipment: (shipmentId) => addToQueue('book', { shipmentId }),
    getTracking:  (shipmentId) => addToQueue('track', { shipmentId }),
    cancelShipment: (shipmentId) => addToQueue('cancel', { shipmentId }),
};
```

### Код `provider.js` (HTTP-клиент)

```javascript
const axios = require('axios');

module.exports = {
    bookShipment: (requestData) => axios.post(
        config.get('dhl.apiUrl') + '/shipments',
        requestData,
        { headers: { 'DHL-API-Key': config.get('dhl.apiKey') } }
    )
};
```

## Как добавить новую интеграцию

1. Создай папку `app/services/integration/[carrier]/`
2. Создай `service.js` — только добавление в очередь
3. Создай `impl.js` — оркестрация (загрузка → трансформация → запрос → сохранение)
4. Создай `dataResolver.js` — загрузка данных из БД
5. Создай `dataBuilder/` — трансформация данных в формат API
6. Создай `provider.js` — HTTP-клиент
7. Зарегистрируй handler в `worker/index.js`
8. Инициализируй event-handlers в `app/server.js` и `worker/index.js`

## Повторные попытки (Retry Logic)

Каждая интеграция имеет собственную стратегию retry:

```javascript
// Пример: app/lib/retryAttempts.js
const retryAttempts = (job, error, maxAttempts, retryDelays) => {
    const attempts = job.data.attempts || 0;
    if (attempts >= maxAttempts) return false;
    const delayMs = retryDelays[attempts] * 1000;
    job.failed().delay(delayMs).state('delayed').priority('high').update();
    job.data.attempts = attempts + 1;
    return true;
};
```

Специфичные методы: `retryAttemptsTeliaJob`, `retryAttepmtsTerrialJob`, `retryAttepmtsWebhookJob`, `retryAttepmtsHeppnerJob`.

## Логирование интеграций

Логи интеграций хранятся в S3:

```javascript
const { buildLogger } = require('../../lib/log');
const logger = buildLogger('integration-dhl');

logger.info('Booking shipment', { shipmentId, carrier: 'DHL' });
logger.error('API error', { error, shipmentId });
```

Логи доступны через `integration_logs_s3_helper.js`.
