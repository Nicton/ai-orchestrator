## Ключевые сервисы

### `resolvers.js` — фундамент
- Базовые функции загрузки сущностей из БД
- Используется 50+ другими сервисами
- Актуален для любого запроса к данным

### `shipments.js` — главный домен
- Самый крупный сервис, тяжёлая координация доменов
- Создание, обновление, отмена перевозок
- Запуск интеграций, отслеживания, уведомлений

### `slots/index.js` — тайм-слоты
- Сложная логика: резервирование, конфликты, права доступа
- Связан с carriers, shippers, shipments

### `accounts.js` — мультиарендность
- Контекст аккаунта, переключение аккаунтов
- ACL-контроль на уровне аккаунта

## Типы сервисов

### 1. Бизнес-логика (крупные файлы)

```javascript
// Пример: app/services/shipments.js
const db = require('../models')();
const logger = buildLogger('shipments');

module.exports = {
    findShipment,
    listShipments,
    createShipment,
    updateShipment,
    cancelShipment,
    // ... 20-50 функций
};
```

### 2. Repository-сервис

```javascript
// app/services/resolvers.js — загрузка сущностей из БД
module.exports = {
    loadShipment,
    loadUser,
    loadCarrier,
    // ...
};
```

### 3. Интеграционный сервис (строгая структура)

```
app/services/integration/[carrier]/
├── service.js       — клиент очереди (точка входа)
├── impl.js          — оркестрация
├── dataResolver.js  — загрузка данных из БД
├── dataBuilder/     — трансформация в формат перевозчика
├── provider.js      — HTTP-клиент к API
├── constants.js     — константы
└── tasks.js         — cron-задачи
```

## Иерархия зависимостей

```
Высокий уровень: API/интеграционные сервисы
    │  (координация, внешние вызовы)
    ▼
Средний уровень: Бизнес-сервисы
    │  (бизнес-правила, стейт-машины)
    ▼
Низкий уровень: resolvers.js, helpers
    │  (доступ к БД, утилиты)
    ▼
Models (Sequelize)
    ▼
PostgreSQL
```

**Пример цепочки зависимостей:**
```
shipments.js
  → resolvers.js        (запросы к БД)
  → attachments/        (вложения)
  → tracking/           (трекинг)
  → integration/dhl/    (внешняя интеграция)
  → orders.js           (связанные заказы)
```

## Middleware-сервисы

Используются для auth/авторизации:

```javascript
// Пример: защита маршрута ролями
const authByRolesMiddleware = roles => (req, res, next) => {
    if (!req.user.roles_list.some(r => roles.includes(r))) {
        return res.status(403).send('Forbidden');
    }
    next();
};
```

## Как найти код по фиче

1. **Эндпоинт известен** → смотри `app/routes/` → контроллер → сервис
2. **Фича с определённым перевозчиком** → `app/services/integration/[carrier]/`
3. **Фоновая задача** → `worker/tasks/` или `app/cron/`
4. **Бизнес-логика** → ищи в `app/services/` по домену
5. **Модель данных** → `app/models/` по имени сущности
