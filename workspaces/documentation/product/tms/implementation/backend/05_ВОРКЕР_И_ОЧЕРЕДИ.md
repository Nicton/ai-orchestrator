# Воркер и очереди

> Источник: `doc/queue-worker-system.md`, `worker/`, `app/cron/`

## Обзор

- **Библиотека**: Kue (Redis-backed)
- **Клиент очереди**: `app/lib/queue.js`
- **Worker**: `worker/index.js`
- **40+ очередей** для разных типов задач

## Запуск воркера

```bash
# Обработка всех очередей
npm run worker

# С авто-перезагрузкой
npm run worker:watch

# Только конкретная очередь
node worker/index.js queueName

# Все кроме указанных
node worker/index.js --ex queueName1,queueName2
```

## Постановка задачи в очередь (из сервиса)

```javascript
const queue = require('../lib/queue');

// Стандартный паттерн
const addToQueue = (task, params) => new Promise((resolve, reject) => {
    queue.create('queueName', { task, params })
        .save(err => err ? reject(err) : resolve());
});

// Использование
addToQueue('sendEmail', { userId: 123, templateId: 'status-changed' });

// Отложенная задача (через 60 минут)
queue.addDelayedTask('processData', { id: 123 }, 60);
```

## Структура handler'а

```javascript
// worker/tasks/example.js
module.exports = (task, params, jobId) => {
    switch (task) {
        case 'taskName1': return handleTask1(params);
        case 'taskName2': return handleTask2(params);
        default: return Promise.reject(new Error(`Unknown task: ${task}`));
    }
};

function handleTask1(params) {
    // Загрузка → Обработка → Сохранение
    return db.Shipment.findById(params.shipmentId)
        .then(shipment => processShipment(shipment))
        .then(result => saveResult(result));
}
```

## Жизненный цикл задачи

```
1. Создание задачи (сервисный слой)
        ↓
2. Очередь Redis (Kue)
        ↓
3. Worker забирает задачу
        ↓
4. Выполнение handler'а
        ↓
   ┌─ Успех ──────► Задача удалена из очереди
   │
   └─ Ошибка
        ↓
        ├─ Retryable ──► Задержка + повторная попытка
        └─ Fatal     ──► Помечена как failed
                          ↓
                         Уведомление (Telegram, Email)
```

## Retry-логика

```javascript
// app/lib/retryAttempts.js
const retryAttempts = (job, error, maxAttempts, retryDelays) => {
    const attempts = job.data.attempts || 0;
    if (attempts >= maxAttempts) return false; // Лимит исчерпан

    const delayMs = retryDelays[attempts] * 1000;
    job.failed().delay(delayMs).state('delayed').priority('high').update();
    job.data.attempts = attempts + 1;
    return true;
};
```

Специфичные стратегии для каждой интеграции:
- `retryAttemptsTeliaJob` — своя задержка, лимит
- `retryAttepmtsTerrialJob`
- `retryAttepmtsWebhookJob`
- `retryAttepmtsHeppnerJob`

const queue = require('./lib/queue');

// Создать задачу

// Обработать задачи

## Как добавить новую background-задачу

1. Определить: нужна ли новая очередь или подходит существующая
2. Создать handler в `worker/tasks/newTask.js`
3. Зарегистрировать в `worker/index.js`:
   ```javascript
   const handlers = {
       // ...
       myNewQueue: require('./tasks/newTask'),
   };
   ```
4. Создать функцию постановки в очередь в нужном сервисе
5. Если нужен cron — добавить файл в `app/cron/`

## Диагностика

**Зависшие задачи:**
```javascript
queue.removeCompletedTasks('queueName', 0);
queue.removeFailedTasks('queueName', 0);
```

**Большая очередь:**
- Проверить, запущен ли воркер
- Увеличить concurrency
- Оптимизировать handler
- Добавить ещё один инстанс воркера

**Задача не обрабатывается:**
- Проверить совпадение имени очереди
- Проверить регистрацию handler'а
- Убедиться что воркер запущен с нужными аргументами
- Проверить подключение к Redis
