# Rate Sheet — Структура и алгоритм сборки

> Источник требований: REQ-RS-001..012 | Слайды: 2024 04 - Rate Sheet structure, 2024 12 - Sea RS

---

## Концепция

Rate Sheet (RS) — матрица тарифов. Каждый RS покрывает один тип услуги (Pre-carriage, Freight, THC и т.д.) и один транспортный режим. Из набора RS собирается **полная стоимость** отправки.

---

## Активация LOCODE для RS (REQ-RS-001)

**LOCODE** (UN/LOCODE) — стандартный код для морских/ж/д/авиа портов.

| Транспортный режим | LOCODE обязателен |
|-------------------|------------------|
| SEA, RAIL, AIR, RORO, River | Да |
| ROAD | Нет (Country + LogZone достаточно) |

При активации LOCODE в RS:
- Поле «PORT» заменяет поле «Country» как приоритетный идентификатор
- LOCODE имеет приоритет над Country при поиске совпадения

---

## Типы услуг (SRS) — 10 категорий (REQ-RS-002)

| Тип | Описание |
|-----|----------|
| **Pre-carriage** | Транспортировка до порта отправления |
| **Freight** | Основной фрахт |
| **Post-carriage** | Транспортировка от порта прибытия |
| **Export Custom** | Таможенное оформление при отправке |
| **Import Custom** | Таможенное оформление при прибытии |
| **THC Origin** | Terminal Handling Charge в порту отправки |
| **THC Destination** | Terminal Handling Charge в порту прибытия |
| **Origin Charges** | Сборы в порту отправки |
| **Destination Charges** | Сборы в порту прибытия |
| **Lead Time** | Структура времени транзита (без стоимости) |

---

## Алгоритм сборки тарифа (REQ-RS-003)

6 шагов сборки (в порядке приоритета):

1. **Фильтр по режиму** — отбрасываются RS с несовпадающим Transport Mode
2. **Фильтр по типу услуги** — отбираются RS с нужным SRS-типом
3. **Geo-matching** — проверка совпадения Origin и Destination (LOCODE → Country → LogZone)
4. **Date validity** — проверка дат действия RS
5. **Entity/Account** — если задан фильтр по аккаунту или entity
6. **Specificities match** — если в RS заданы специфики (DGD, Dangerous, etc.)

Первый RS, прошедший все 6 шагов, — победитель. Если нет совпадения → «No rate found».

**Правило приоритета Geo:** LOCODE > Country > LogZone при перекрытии.

---

## Правила импорта и валидации RS (REQ-RS-004)

При загрузке CSV/Excel-файла RS:

| Проверка | Действие при ошибке |
|----------|-------------------|
| Дублирующие строки (Origin + Dest + Mode + Date) | Ошибка, загрузка блокируется |
| Отсутствие обязательных полей | Подсветка строки |
| Неверный LOCODE | Предупреждение (не блокирующее) |
| Пересекающиеся диапазоны дат | Ошибка |

Успешная загрузка → RS статус ACTIVE. Старые строки → ARCHIVED.

## Округление весового коэффициента (WM) (REQ-RS-005)

| Значение WM | Точность округления |
|-------------|-------------------|
| 1 | До целого |
| 0.1 | До десятых |
| 0.01 | До сотых |

WM (Weight/Measurement ratio) используется для расчёта платного веса:
`Chargeable Weight = max(Actual Weight, Volume × WM)`

---

## Опасные грузы (DGD) в RS (REQ-RS-006)

| Поле | Описание |
|------|----------|
| DGD Toggle | NO / YES |
| UN CODE | Обязателен, если DGD = YES и задан конкретный класс |
| Packing Group | I / II / III |
| Class | 1–9 |

Матчинг DGD:
- RS без DGD-фильтра → принимает любой груз
- RS с DGD=YES без UN CODE → принимает все DGD грузы
- RS с DGD=YES + UN CODE → только для конкретного класса DGD

---

## Структура Lead Time (REQ-RS-007)

Lead Time RS содержит:

| Поле | Описание |
|------|----------|
| Transit Days | Количество транзитных дней |
| Cut-off | День недели + время (для порта/терминала) |
| Departure Day | Фиксированный день отправки |
| ETA | Расчётная дата прибытия |

При наличии Lead Time RS → ETA рассчитывается автоматически при бронировании.

---

## Мультиконтейнерные RS (REQ-RS-008)

Для Multi Container отправлений:

- Каждый тип контейнера (20DV, 40HC, 45HC и т.д.) → отдельная строка тарифа в RS
- Если тип контейнера не найден в RS → используется строка с типом «ALL» (если задана)
- Расчёт: стоимость × количество контейнеров данного типа

**Shiptify Internal Key** — маппинг от кода перевозчика к стандартному типу контейнера Shiptify:

| Пример кода перевозчика | Тип Shiptify |
|------------------------|-------------|
| 20G1, 22G1 | 20DV |
| 40HC, 42G1 | 40HC |
| 45G1, L5G1 | 45HC |

---

## Влияние инкотермов на RS (REQ-RS-009)

| Инкотерм | Origin Charges | Destination Charges |
|---------|----------------|---------------------|
| EXW, FCA | Не включены | Включены |
| DAP, DDP | Включены | Не включены |
| CFR, CPT | Включены | Зависит |
| FOB | Origin на продавца | Destination на покупателя |

При BUY & SELL: инкотерм определяет, какой набор SRS-типов применяется для BUYING и SELLING сторон.

---

## Шаблон RS (REQ-RS-010)

Кнопка «Download Template» → Excel-шаблон с:
- Заголовки колонок по выбранному типу услуги
- Пример заполненной строки
- Лист «LOCODE Reference» (если режим поддерживает LOCODE)
- Лист «Packing Groups» для DGD RS

Шаблон зависит от: транспортного режима + типа услуги.

---

## LOCODE-поиск (REQ-RS-011)

- Автодополнение при вводе кода или названия города/порта
- База содержит UN/LOCODE для 180+ стран
- При отсутствии LOCODE → ввод страны + порта вручную
- Ненайденный LOCODE → предупреждение, но не блокирует сохранение

---

## Лимиты и ограничения RS (REQ-RS-012)

| Параметр | Ограничение |
|----------|------------|
| Максимум строк в одном RS | 10 000 |
| Максимальный размер файла импорта | 10 MB |
| Количество активных RS на аккаунт | Без ограничений |
| Дата действия | Обязательна; RS без даты окончания = бессрочный |
