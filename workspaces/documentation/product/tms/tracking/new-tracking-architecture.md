# New Tracking Architecture — Новая архитектура трекинга

Новая 3-экранная архитектура страниц трекинга с раздельными настройками для Buyer и режима транспортировки.

> Источник: слайд `2026 04 - NEW TRACKING`

---

## 3 экрана трекинга

```
1. Setup Screen         — настройки отображения
2. Shipment Screen      — внутренняя страница (для Shipper/Carrier)
3. Public Tracking Page — публичная страница (отдельно от внутренней)
```

Публичная и внутренняя страницы **разделены** и имеют независимые конфигурации.

---

## 3-уровневый fallback для настроек

| Уровень | Условие | Поведение |
|---------|---------|----------|
| **No setup** | Нет настроек | Текущий экран без изменений |
| **Per-Buyer** | Настройка для конкретного Buyer | Buyer-специфичный вид |
| **Per-Mode** | Настройка по режиму | Mode-специфичный вид с фиксированными show/hide полями |

---

## Сортировка

| Страница | Порядок |
|---------|---------|
| Public page | Новые события сверху (newest-first) |
| Internal page | Старые события сверху (oldest-first) |

---

## Exception Flags

Флаги исключений на кодах трекинга → оранжевые предупреждения на связанных задачах процесса.

---

## Статус-бейджи (публичная страница)

| Бейдж | Вид | Значение |
|-------|-----|---------|
| **"At Risk"** | Пульсирующий | Перевозка под риском задержки |
| **"On Time"** | Статичный | Всё по расписанию |

**Predicted Delivery ETA** показывается с перечёркнутым исходным SLA:
```
Ожидаемая доставка: 15 июня 14:00
~~Плановая: 14 июня 18:00~~
```
