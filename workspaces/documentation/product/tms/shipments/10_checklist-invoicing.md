## INVOICING


### REQ-INV-001 — Фильтрация строк инвойса по статусу PENDING перед созданием пре-инвойса

**Приоритет:** P0 | **Источник:** 2023 05 - Invoicing Process - Google Slides.txt, Page 14


**Проверки:**
- [ ] На экране листинга инвойс-строк по умолчанию применён фильтр статуса = PENDING
- [ ] Пользователь не может выбрать строки с другими статусами для генерации пре-инвойса
- [ ] Количество выбранных строк отображается в заголовке кнопки 'Generate X Pre-Invoices'

### REQ-INV-002 — Массовый выбор строк: чекбокс 'Select All' в заголовке таблицы

**Приоритет:** P0 | **Источник:** 2023 05 - Invoicing Process - Google Slides.txt, Page 14; 2026 01 - Invoice Improvement, Page 4


**Проверки:**
- [ ] В заголовке таблицы присутствует кнопка SELECT ALL
- [ ] При нажатии SELECT ALL все видимые строки со статусом PENDING выбираются
- [ ] При снятии SELECT ALL все строки снимаются с выбора

### REQ-INV-003 — Генерация пре-инвойсов: 1 пре-инвойс на партнёра

**Приоритет:** P0 | **Источник:** 2023 05 - Invoicing Process - Google Slides.txt, Page 14-17


**Проверки:**
- [ ] Если пользователь авторизован как Carrier — система создаёт 1 пре-инвойс на каждого уникального Shipper среди выбранных строк
- [ ] Если пользователь авторизован как Shipper — система создаёт 1 пре-инвойс на каждого уникального Carrier среди выбранных строк
- [ ] После подтверждения система показывает список созданных пре-инвойсов с именем, суммой и количеством букингов

### REQ-INV-004 — Кнопка 'Generate Invoices' для аккаунтов SELLER с включённой опцией Invoicing Creation

**Приоритет:** P1 | **Источник:** 2023 05 - Invoicing Process - Google Slides.txt, Page 14

- [ ] При значении BO/Settings/Invoicing Creation = NO кнопка генерации инвойсов не отображается
- [ ] Количество в кнопке корректно отражает число выбранных строк

### REQ-INV-005 — Имя пре-инвойса: формат PI + год + месяц + 6-значный инкрементальный номер

- [ ] Имя генерируется по шаблону: PI [YYYY] [MM] - [000001]
- [ ] Инкремент является глобальным на уровне Shiptify (не на уровне аккаунта)
- [ ] В диалоге подтверждения отображаются сгенерированные имена до фактического создания

### REQ-INV-006 — Период отображения пре-инвойсов: последние 2 месяца + текущий месяц

**Приоритет:** P1 | **Источник:** 2023 05 - Invoicing Process - Google Slides.txt, Page 16


**Проверки:**
- [ ] При открытии раздела Pre-Invoices отображаются записи за текущий месяц и 2 предыдущих
- [ ] Пре-инвойсу не назначается месяц автоматически (поле 'Assigned Month' не заполняется при создании)
- [ ] Пользователь может вручную назначить месяц позднее

### REQ-INV-007 — 7 статусов пре-инвойса/инвойса: DRAFT, PENDING, TO CHECK, BLOCKED, CHECKED, VALIDATED, CANCELLED

**Приоритет:** P0 | **Источник:** 2023 05 - Invoicing Process - Google Slides.txt, Page 26-28


**Проверки:**
- [ ] Система поддерживает ровно 7 статусов для пре-инвойса: DRAFT, PENDING, TO CHECK, BLOCKED, CHECKED, VALIDATED, CANCELLED
- [ ] Статус DRAFT виден только создателю (CREATOR), недоступен для CHECKER
- [ ] Переходы статусов соответствуют матрице: DRAFT→PENDING (SHARE), PENDING→VALIDATED/BLOCKED (CREATOR), PENDING→CHECKED (CHECKER), BLOCKED→PENDING (UNBLOCK), CHECKED→VALIDATED/BLOCKED (CREATOR), CHECKED→PENDING (UNCHECK)

### REQ-INV-008 — Ролевые ограничения действий по статусам для CREATOR и CHECKER

**Приоритет:** P0 | **Источник:** 2023 05 - Invoicing Process - Google Slides.txt, Page 26-28


**Проверки:**
- [ ] В статусе DRAFT: кнопки действий отсутствуют у обоих; у CREATOR есть кнопка SHARE
- [ ] В статусе PENDING/TO CHECK: CREATOR видит VALIDATE и BLOCK; CHECKER видит CHECKED
- [ ] В статусе BLOCKED: CREATOR видит UNBLOCK; у CHECKER нет действий
- [ ] В статусе CHECKED: CREATOR видит VALIDATE и BLOCK; CHECKER видит UNCHECK
- [ ] В статусе VALIDATED: у SELLER отображается CREATE Invoice
- [ ] Кнопка CANCEL доступна во всех статусах и снимает назначение всех строк с пре-инвойса

### REQ-INV-009 — Видимость пре-инвойса по статусам для обеих сторон

**Приоритет:** P1 | **Источник:** 2023 05 - Invoicing Process - Google Slides.txt, Page 26-27


**Проверки:**
- [ ] Статус DRAFT: пре-инвойс видим только CREATOR, CHECKER не видит его в своём списке
- [ ] Статусы PENDING, TO CHECK, BLOCKED, CHECKED, VALIDATED, CANCELLED: пре-инвойс виден обеим сторонам (CREATOR и OTHER SIDE)

### REQ-INV-010 — Отмена пре-инвойса: снятие назначения всех строк

**Приоритет:** P0 | **Источник:** 2023 05 - Invoicing Process - Google Slides.txt, Page 28

### REQ-INV-012 — Список пре-инвойсов: отображение сводной информации и счётчиков по статусам строк

### REQ-INV-014 — Статусы инвойсинга строки: NOT PRICED, TO CHECK, GAP, BLOCKED, CHECKED

**Приоритет:** P0 | **Источник:** 2023 05 - Invoicing Process - Google Slides.txt, Page 10


**Проверки:**
- [ ] Система поддерживает 5 статусов инвойсинга для строки: NOT PRICED, TO CHECK, GAP, BLOCKED, CHECKED
- [ ] Статус GAP (ранее Gap Analysis Ongoing) отображается с меткой 'écart' во французской локали
- [ ] Статус TO CHECK (ранее Waiting for Invoice) отображается с меткой 'à contrôler' во французской локали
- [ ] Статусы Invoiced и Closed из старой версии маппятся на CHECKED

### REQ-INV-015 — Статус инвойсирования транспорта: Pending, Pre-invoiced, Invoiced, No Invoice, Cancelled

**Приоритет:** P0 | **Источник:** 2023 05 - Invoicing Process - Google Slides.txt, Page 12, 50


**Проверки:**
- [ ] На уровне листинга присутствует отдельный фильтр 'Invoicing' (FR: Facturation) со значениями: Pending, Pre-invoiced, Invoiced, No Invoice, Cancelled
- [ ] Исторические записи: если старый статус = CHECKED, INVOICED или CLOSED → новый статус = Invoiced; иначе → Pending
- [ ] После назначения строки на пре-инвойс статус строки обновляется до Pre-invoiced

### REQ-INV-017 — Создание инвойса вручную: обязательные поля
