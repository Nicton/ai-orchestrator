# Типы бронирования — Booking Types

> Источник требований: REQ-BOOK-012..016 | Слайды: 2026 01 - Flow Types

---

## Обзор

**Тип бронирования** определяет, каким образом заявка назначается перевозчику. Это второй ключевой выбор шиппера после выбора типа создания заявки (см. [flow-types.md](flow-types.md)).

| Тип | Код | Участие человека | Перевозчиков | Авто-назначение |
|-----|-----|-----------------|--------------|-----------------|
| Direct Booking | **DB** | Шиппер выбирает | 1 | Нет |
| Quote Request | **QR** | Шиппер выбирает котировку | N | Нет |
| Automated Direct Booking | **ADB** | Нет (полностью авто) | 1 | Да |
| Semi Automated Direct Booking | **SADB** | Шиппер выбирает перевозчика | 1 | Нет |
| First Responder | **FR** | Нет (первый с ценой &lt;= лимит) | N | Да |

Все типы бронирования поддерживаются для режимов: **Air / Sea / Road**.

---

## Direct Booking (DB) — REQ-BOOK-012

Шиппер вручную выбирает одного перевозчика из панели и отправляет заявку напрямую.

**Процесс:**
1. Шиппер выбирает перевозчика из списка
2. Перевозчик получает уведомление
3. Перевозчик нажимает **Green** (подтвердить) или **Red** (отклонить)
4. При подтверждении SR переходит в статус `confirmed`; при отклонении — `refused`

**Ключевые правила:**
- Перевозчик без доступа к заявке не видит её в своём листинге
- Шиппер может добавить комментарий (поле появляется только при выбранном перевозчике — REQ-BOOK-017)
- Поддерживается как ручное создание (SC/TC), так и через TRC (Buy&Sell)

---

## Quote Request (QR) — REQ-BOOK-013

Шиппер рассылает запрос на котировку нескольким перевозчикам одновременно.

**Процесс:**
1. Шиппер выбирает N перевозчиков
2. Каждый перевозчик видит запрос и отправляет котировку в стандартизированном формате
3. Шиппер получает единый интерфейс сравнения всех котировок
4. Шиппер выбирает победившую котировку — процесс продолжается как Direct Booking

**Ключевые правила:**
- Каждый перевозчик может отправить только одну котировку (если не включена опция `multiple quotes per carrier`)
- QR без точных дат сохраняется как черновик
- Дедлайн ответа задаётся шиппером при отправке QR

**Стратегии котировок (REQ-RS-017):** см. дополнительные стратегии — Buy-It-Now, Hybrid, Live Reverse Auction, Automated Sealed Bid — в документе [retro-consolidation.md](../features/retro-consolidation.md).

## Automated Direct Booking (ADB) — REQ-BOOK-014

Полностью автоматизированное бронирование: ERP/WMS создаёт заявку и отправляет её перевозчику без участия шиппера.

**Процесс:**
1. ERP/WMS отправляет данные через API
2. Система автоматически назначает перевозчика (по Transport Plan / Rate Management)
3. Перевозчик уведомляется и подтверждает/отклоняет кнопками Green/Red

**Ключевые правила:**
- Выбор перевозчика управляется настройками Transport Plan и Rates Management
- Ошибки API логируются в Business Log с payload (REQ-BOOK-031)
- Шиппер не участвует в процессе — только получает результат

---

## Semi Automated Direct Booking (SADB) — REQ-BOOK-015

Гибридный режим: ERP/WMS создаёт черновик, а шиппер вручную выбирает перевозчика перед отправкой.

**Процесс:**
1. ERP/WMS создаёт заявку со статусом `DRAFT`
2. Шиппер получает уведомление о новом черновике
3. Шиппер просматривает данные, при необходимости дополняет, прикрепляет документы
4. Шиппер выбирает перевозчика и отправляет заявку
5. Перевозчик уведомляется и подтверждает/отклоняет

**Ключевые правила:**
- Заявка остаётся в статусе `DRAFT` до ручного завершения шиппером
- Шиппер видит список всех SADB-черновиков с возможностью фильтрации
- Подходит для случаев, когда данные из ERP требуют проверки перед отправкой

---

## First Responder (FR) — REQ-BOOK-016

Гонка между перевозчиками: первый, кто предложит цену ниже установленного лимита, автоматически получает заявку.

**Процесс:**
1. Шиппер задаёт **максимальную стоимость** (Target Price) — публичную или приватную
2. Шиппер выбирает несколько перевозчиков
3. Перевозчики конкурируют, отправляя котировки
4. Первый перевозчик с ценой `&lt;= Target Price` **автоматически получает заявку**
5. Остальные перевозчики уведомляются об отмене запроса

**Ключевые правила:**
- Если ни один перевозчик не уложился в лимит за заданное время — заявка остаётся открытой для ручного выбора
- Шиппер видит количество перевозчиков, успевших ответить до назначения
- Target Price может быть скрытой от перевозчиков (они не знают точный лимит)

## Выбор типа бронирования

| Сценарий | Рекомендуемый тип |
|----------|------------------|
| Создание через API, шиппер выбирает перевозчика вручную | **SADB** |
| Создание через API, полностью автоматическое назначение | **ADB** |
| Ручное создание, нужны котировки от нескольких перевозчиков | **QR** |
| Ручное создание, один конкретный перевозчик | **DB** |
| Гонка перевозчиков с ценовым лимитом | **FR** |
