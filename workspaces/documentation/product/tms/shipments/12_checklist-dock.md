### REQ-DOCK-001 — Создание нового экрана «Dock Door» с назначением слотов на ворота

**Приоритет:** P0 | **Источник:** 2026 05 - Dock Door assignment screen - Google Slides.txt


**Проверки:**
- [ ] Добавить новый раздел «Dock Door» (строковый ключ + иконка)
- [ ] Экран доступен для всех DOCK-аккаунтов (уточнить: опция BO или по умолчанию)
- [ ] Временная сетка: ось X — время (в рамках часов открытия зоны), ось Y — ворота (Dock Doors)
- [ ] Лоток неназначенных слотов отображается внизу экрана
- [ ] Drag & drop: из лотка на ворота и между воротами; перепланировка времени с этого экрана недоступна
- [ ] Карточка слота автоматически позиционируется на плановое время

### REQ-DOCK-002 — Отображение карточек слотов на экране Dock Door

**Приоритет:** P0 | **Источник:** 2026 05 - Dock Door assignment screen - Google Slides.txt

### REQ-DOCK-021 — Структура Partner DB: роли SELLER / BUYER / BOOKER / SPECTATOR

**Приоритет:** P1 | **Источник:** 2025 05 - SLOT MULTI ORDER MULTI CUSTOMER - Google Slides.txt


**Проверки:**
- [ ] INBOUND: SELLER ID = Supplier ID, BUYER ID = Customer ID, BOOKER ID — опционально
- [ ] OUTBOUND: SELLER ID = Customer ID, BUYER ID = Supplier ID, BOOKER ID — опционально
- [ ] Если ID заполнен — игнорировать введённое вручную имя; имя без ID — только отображение
- [ ] Видимость привязана к слоту (snapshot), не ретроактивна
- [ ] Логотип партнёра наследуется из аккаунта, если не загружен вручную
- [ ] Перенести VAT под поле location в карточке партнёра

- [ ] Созданный пользователь — статус pending, обогащает carrier contacts (не как invite)
