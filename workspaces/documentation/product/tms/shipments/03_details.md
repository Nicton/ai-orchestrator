# Страница деталей перевозки

## Что это

Полная карточка одной перевозки. Организована в табы — каждый таб отвечает за свою область жизненного цикла. Это главная рабочая страница для Carrier (обновление трекинга) и Shipper (контроль статуса и документов).

**URL:** `/shipments/{id}`
**Frontend:** `workspaces/frontend/public/app/shipments/view/`

---

## Табы

### Таб 1: Tracking (Трекинг) — по умолчанию

Список трекинг-точек: каждая точка — это событие в жизни перевозки (забор груза, прибытие на склад, доставка).

#### Что видит пользователь

| Элемент | Описание |
|---------|---------|
| Трекинг-точки | Список: тип (pick-up / arrival), локация, плановая дата, статус |
| Статус точки | Not confirmed / Confirmed / Delayed |
| Кнопки на точке | Подтвердить / Перепланировать / Запросить инфо |
| Subcontract | Данные субподрядчика (для Road, для Carrier) |
| Интеграции | Статус активных интеграций (DHL, FedEx и др.) + сообщения об ошибках |

#### Действия

| Действие | Что происходит | Кто может |
|----------|---------------|-----------|
| Подтвердить точку | `TrackingPoint.status = confirmed`, пересчёт статуса Shipment | Carrier |
| Перепланировать точку | Открывает форму изменения даты | Carrier, Shipper |
| Запросить инфо | Отправляет email перевозчику | Shipper |
| Добавить трекинг-точку | Открывает форму `/shipments/{id}/track` | Carrier |
| Отменить перевозку | `Shipment.status = canceled` + интеграции отмены | Shipper, Admin |
| Реактивировать | Восстанавливает отменённую перевозку | Admin |
| Запросить POD | Отправляет запрос Proof of Delivery | Shipper |
| Загрузить документ | Drag-and-drop в правую панель | Carrier, Shipper |
| Расшарить трекинг | Генерирует публичную ссылку на трекинг | Shipper |
| Связаться | Открывает чат или email | Все |

## Мутации

### Подтверждение трекинг-точки

**Внутренние:**
- `TrackingPoint.status` → `confirmed`
- `TrackingPoint.real_date` = текущая дата
- `Shipment.status` пересчитывается через `calculateShipmentStatus()`

**Внешние:**
- Email `mailStatus` → shipper + подписчики
- Webhook `shipmentStatusUpdated`
- Если интеграция активна: обновление в системе перевозчика (P44, Shippeo и др.)

### Отмена перевозки

**Внутренние:**
- `Shipment.status` → `canceled`
- `TrackingPoint[]` — помечаются как отменённые

**Внешние:**
- Carrier API: отмена бронирования через `runCancelShipmentIntegrations()`
- Email `mailShipmentCanceledToCarrier` → перевозчику
- Email `mailShipmentCanceledToShipper` → shipper'у
- Webhook `shipmentCancelled`

### Загрузка документа

**Внутренние:**
- `ShipmentAttachment` создаётся
- Если это POD: `Shipment.pod_status` обновляется

**Внешние:**
- AWS S3: файл загружается в бакет
- Email `mailAddAttachment` → уведомление о новом документе

## Переходы

- "← Назад" → `/shipments` (список)
- Перевозчик (клик) → `/partners/{id}` или `/carriers/{id}`
- Кнопка "Слот" → `/slots/{id}` (связанный слот)
- Заказ (в табе Orders) → `/orders/{id}`
- Претензия → `/claims/{id}`
