## 3. Исправленный Qase-кейс 1340 (готов к замене)

**Название:** RTB: после создания заявка получает статус Ready to Book и видна на вкладке «Ready to book»

**Предусловия:** пользователь Shipper с правом создания SR; CSW доступен.

**Шаги:**
1. Создать заявку через CSW, на финальном шаге выбрать «Ready to Book» (не Draft).
2. Открыть booking list.
3. Открыть вкладку **«Ready to book»**.

**Ожидаемый результат:**
- Заявка имеет статус `ready_to_book` (отображение «Ready to book», синяя метка) — НЕ «Draft»;
- Заявка отображается на вкладке «Ready to book»; на вкладке «Draft» отсутствует;
- Доступные переходы: → sent_to_carrier (бронирование) / отмена.

**Примечание:** прежняя формулировка «включается фильтр Draft» — copy-paste из кейса 1339; статусы `new` (Draft) и `ready_to_book` — раздельные (код: `shipment_request.js:718,825`).

## 4. Предложения по улучшению документации (опционально)

1. **Frontmatter с Confluence-ID** в каждом md (`confluence_id: 6225...`) — публикация по ID вместо поиска по заголовку (исключит инциденты типа «Claims → не та страница»).
2. **CI-проверка ссылок** — скрипт линк-чека (уже написан) повесить на pre-commit/CI репозитория documentation.
3. **Единый publish-скрипт** — консолидировать ~10 однотипных `scripts/publish-*.mjs` в один с конфигом (mdToStorage уже идентичен).
4. **Док-долг по интеграциям** — у 26 перевозчиков детальные доки есть у ~10; шаблон per-carrier (протокол, конфиг-ключи, cron, маппинг STY) закроет остальные.
5. **Свежесть**: в статус-баннерах есть даты; раз в квартал — повторный мини-аудит «реализовано/нет» (особенно после релиза Quote Strategy TMS-3557).
6. **RU/EN**: сейчас докуменация RU с EN-терминами — для внешних партнёров может потребоваться EN-слой (решение Product).
