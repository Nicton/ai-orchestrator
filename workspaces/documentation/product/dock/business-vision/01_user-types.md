# Типы пользователей — Dock модуль

Dock-модуль обслуживает несколько категорий пользователей с разными правами и сценариями работы.

---

## Оператор / Dock Manager (Управляющий воротами)

**Кто это:** Сотрудник склада или терминала, ответственный за управление воротами.

**Цель:** Контролировать поток транспорта, назначать ворота, подтверждать и переносить слоты.

**Что делает в системе:**
- Просматривает планирование на день, неделю (Board / Day / Week views)
- Подтверждает или отклоняет запросы на слоты
- Назначает конкретные Dock Doors через drag-and-drop
- Обновляет статус слота в реальном времени (driver_called → on_site → at_dock → loaded)
- Настраивает зоны и расписание (Zone Calendar)
- Управляет повторяющимися (recurring) слотами

**Тип аккаунта в системе:** `Dock` account  
**ACL ключ:** `requireShipper` / `loadOwnPmlSlot`

**Доступные представления:**

| Вид | URL |
|-----|-----|
| Board | `/slotify/board` |
| Week | `/slotify/week` |
| Day | `/slotify/load-day` |
| Door Assignment | `/slotify/load/YYYY-MM-DD` |

---

## Shipper (Грузовладелец / Логист)

**Кто это:** Компания, которая управляет складом и создаёт слоты для входящих/исходящих отправок.

**Цель:** Заранее зарезервировать временные окна для своих отправок, управлять нагрузкой склада.

**Что делает в системе:**
- Настраивает Master Locations, зоны (LocationZone), Dock Doors
- Бронирует слоты через основное приложение (SLOT BOOKING card в сайдбаре)
- Запускает SlotBook для бронирования через перевозчика
- Экспортирует данные слотов в Excel
- Настраивает динамическое именование слотов (My Settings → Slot naming)
- Конфигурирует характеристики ворот (Dock Door Specificities)

**Роли внутри Shipper-аккаунта:**

| Роль | Что может |
|------|----------|
| `api` | Стандартный доступ — бронировать слоты, управлять зонами |
| `self_admin` | Управлять пользователями своего аккаунта |
| `spectator` | Только просмотр |

---

## Carrier (Перевозчик)

**Кто это:** Транспортная компания, приезжающая на склад для погрузки/разгрузки.

**Цель:** Зарезервировать временное окно для прибытия на склад.

**Что делает в системе:**
- Бронирует слоты через кнопку "+ BOOK A SLOT" в основном приложении
- Работает с Slotbook (список своих слотов у разных shipper'ов)
- Просматривает историю и предстоящие слоты

**Точки входа:**

| Точка входа | Назначение |
|------------|------------|
| `+ BOOK A SLOT` (синяя кнопка) | Начало бронирования через 6-шаговый визард |
| `/slotbook-ongoing-slots` | Текущие слоты |
| `/slotbook-history-slots` | История слотов |
| `/slotbook-to-be-booked-slots` | Отправки без слота |

---

## PML User / Внешний пользователь Slotify

**Кто это:** Поставщик, экспедитор или перевозчик без аккаунта Shiptify, бронирующий через публичный портал Slotify.

**Цель:** Заполнить данные о грузе и зарезервировать временное окно на конкретный склад.

**Что делает:**
- Переходит по ссылке Slotify (привязана к конкретному PML)
- Проходит 6-шаговый процесс регистрации и бронирования
- Получает подтверждение по email

**Данные сессии:** Бронирование создаётся через `bookSlotifyShipment()` в сервисе `app/services/slotify.js`. Учётная запись создаётся автоматически как pending.

**Поле Department** (только для Supplier/Customer пользователей): Sales Administration, Planning, Purchasing/Procurement, Warehouse & Operations, Transport & Freight, Production/Manufacturing, Finance & Accounting, Maintenance & Facilities, Customer Service, General Management, Other.

---

## Admin (Администратор Shiptify)

**Кто это:** Внутренний сотрудник Shiptify с полным доступом.

**Что делает в Dock:**
- Полный доступ к настройкам зон, слотам, справочникам
- Управление справочником `BO > Dico > Dock-specificities`
- Принудительная смена статусов слотов
- Реактивация отменённых слотов (`reactivateSlots()`)
- Настройка Master Location и Customer zones
