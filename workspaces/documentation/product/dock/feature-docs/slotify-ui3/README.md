# Slotify UI 3.0 / 3.1 — Редизайн публичного портала бронирования

> Источник требований: REQ-SLOTIFY-001..007, REQ-SLOTIFY-013..015 | Слайды: 2026 05 - Slotify UI 3.0, 2026 05 - Slotify UI 3.1

---

## Обзор

Slotify UI 3.x — новый дизайн публичного портала бронирования слотов. Редизайн разбит на два релиза:
- **UI 3.0** — шапка, сводная панель, логотип зоны, финальный экран инструкций
- **UI 3.1** — разделение шага 1 на 1.1 и 1.2, обновлённые формы регистрации, пакинг-лист

---

## UI 3.0 — Изменения шапки и навигации

### Редизайн шапки (REQ-SLOTIFY-001)

| Элемент | Изменение |
|---------|-----------|
| Текст «Slot Booking» | Удалён |
| Адрес | Две строки: адресная строка 1 + 2; отдельно — индекс, город, страна |
| Логотип PML | Увеличен, выровнен с тремя текстовыми строками |
| Ссылка «Contact us» | Ведёт на `contact.slotify@shiptify.com` |

### Сводная левая панель (REQ-SLOTIFY-002)

После нажатия NEXT на любом шаге в левой панели отображаются данные пользователя:

| Поле | Источник |
|------|---------|
| Логотип | Из домена пользователя |
| Email | Введённый email |
| Имя и фамилия | Введённые данные |
| Телефон | Введённый телефон |

- При смене зоны — логотип зоны в левой панели обновляется (заменяет предыдущий)
- Шаги 1.1 и 1.2 сохраняют карту (map) на левой стороне

### Логотип зоны (REQ-SLOTIFY-014)

- При выборе зоны → логотип выбранной зоны появляется в левой сводной панели
- При смене зоны A → зона B: логотип обновляется (не дублируется)

### Страница инструкций (REQ-SLOTIFY-015)

- На финальном шаге (выбор ворот / подтверждение) вместо карты отображается страница **INSTRUCTIONS**
- На ранних шагах инструкции не показываются — только на последнем шаге

## UI 3.1 — Разделение шага регистрации

### Шаг 1.1 и 1.2 (REQ-SLOTIFY-003)

Единый шаг регистрации разбивается на два:

**Шаг 1.1 — Тип и роль:**
- Выбор типа операции: **Deliver** / **Pickup**
- Выбор роли: **Carrier** / **Supplier-Customer**
- Email на этом шаге НЕ запрашивается

**Шаг 1.2 — Контактные данные:**
- Email — **обязателен**
- Телефон — **обязателен**
- Опция «Получить ссылку по email»
- Опция «Войти в аккаунт» — с учётом языка, выбранного на странице Slotify

## Формы регистрации по типу

### Форма CARRIER (REQ-SLOTIFY-004)

| Поле | Условие | Примечание |
|------|---------|-----------|
| Country | Всегда | Флаги стран — flat design, флаг слева |
| City | Только после заполнения Country | — |

**UX-детали:**
- Выпадающий список открывается вниз; поиск встроен в поле
- В поиске городов: сначала результаты с совпадающим началом (alphabetically), затем остальные
- Игнорирование дефиса при поиске: `saint cast` → `saint-cast-le-guildo`
- ISO5-код в результатах поиска не отображается

### Форма Supplier / Customer (REQ-SLOTIFY-005)

| Поле | Описание |
|------|----------|
| Логотип | Логотип домена (или первая буква если логотип не найден) |
| Company | Название компании |
| Name | Имя пользователя |
| Department | Предустановленный список подразделений |

**Список значений Department:**
Sales, Planning, Purchasing, Warehouse, Transport, Production, Finance, Maintenance, Customer Service, General Management, Other

Значение Department сохраняется на уровне DOMAIN при его создании.

**Порядок полей:** Company → Name → Service (Department)

## Пакинг-лист в Slotify (REQ-SLOTIFY-006)

Режим отображения зависит от типа аккаунта:

| Условие | Страница |
|---------|----------|
| PML НЕ является DOCK ORDER и заказов ≥ 2 | **GENERAL PACKING LIST** (не детальный по заказам) |
| Иначе | Детальный пакинг-лист по заказу |

### Правила General Packing List

| Правило | Описание |
|---------|----------|
| TYPE | Автозаполнение первым значением из настроек PML (если General PL не заполнен суммой заказов) |
| Floor: значение | По умолчанию = TOTAL |
| Floor: цвет | Серый если Floor = Total; светло-голубой если Total > Floor |
| Floor: отображение | Переносится на вторую строку |
| Название заказа | Видно на всей строке (без текстов «total» и «floor») |
| Иконка | Для нескольких заказов: не повторяется на 2-й, 3-й строках |

## Данные водителя и грузовика (REQ-SLOTIFY-007)

| Изменение | Описание |
|-----------|----------|
| Метаданные | UI паттерн «Field Name \| значение» на одной строке |
| Группировка | Метаданные и данные водителя/грузовика — выше поля COMMENT |
| INSTRUCTIONS | Отображаются вместо карты на соответствующем шаге |

|---------|----------|
