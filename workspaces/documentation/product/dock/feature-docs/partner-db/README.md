# Partner DB — База данных партнёров

> Источник требований: REQ-DOCK-021..023 | Слайды: 2025 05 - SLOT MULTI ORDER MULTI CUSTOMER

---

## Концепция

**Partner DB** — единый справочник бизнес-партнёров склада (поставщики, покупатели, экспедиторы). Партнёры привязываются к Dock Orders в определённых ролях и управляют видимостью заказов.

---

## Роли партнёров (REQ-DOCK-021)

Каждый партнёр в Dock Order имеет одну из четырёх ролей:

| Роль | Описание |
|------|----------|
| **SELLER** | Продавец груза |
| **BUYER** | Покупатель груза |
| **BOOKER** | Организатор транспортировки (опциональная роль) |
| **SPECTATOR** | Наблюдатель (планируется, не реализовано в текущей версии) |

### Маппинг ролей по направлению

| Направление | SELLER ID | BUYER ID |
|-------------|-----------|----------|
| **INBOUND** | = Supplier ID | = Customer ID |
| **OUTBOUND** | = Customer ID | = Supplier ID |

### Правила заполнения

| Правило | Описание |
|---------|----------|
| ID vs имя | Если ID заполнен — введённое имя игнорируется |
| Имя без ID | Используется только для отображения, не влияет на логику |
| Snapshot | Видимость привязывается к слоту в момент создания, не ретроактивна |
| Логотип | Наследуется из аккаунта партнёра; если не задан — загружается вручную |
| VAT | Поле VAT расположено под полем location в карточке партнёра |

## Видимость Dock Order для партнёров (REQ-DOCK-022)

### Роль BOOKER

| Функция | Доступ |
|---------|--------|
| Просмотр Dock Orders | TRACK / DOCK ORDER |
| Создание слота из DO | ✅ |
| Перепланирование слота | ✅ (как SlotBooker) |
| Отмена слота | ✅ |

### Роль SUPPLIER

| Функция | Доступ |
|---------|--------|
| Статус Dock Order | TRACK — только чтение |
| Видимость слота | Ограниченная — только свои заказы |

### Слот с несколькими клиентами

| Ситуация | Видимость клиенту |
|----------|-----------------|
| SDO принадлежит этому клиенту | Видит слот |
| SDO принадлежат разным клиентам | Не видит слот других клиентов |

> Роль SPECTATOR через PARTNER DB планируется в будущих версиях. Текущая роль TMS Spectator — отдельный механизм.

## Референсы партнёров (REQ-DOCK-023)

Dock Order поддерживает три независимых референса по ролям:

| Поле | Описание |
|------|----------|
| **SELLER REF** | Референс со стороны продавца |
| **BUYER REF** | Референс со стороны покупателя |
| **BOOKER REF** | Референс экспедитора/организатора |
| **EXTERNAL REF** | «Плавающий» референс — только для отображения, не влияет на процесс |

### Поиск заказов

Листинг Dock Orders поддерживает поиск по всем трём ролевым референсам:
- Поиск по SELLER REF
- Поиск по BUYER REF
- Поиск по BOOKER REF

### Уникальность ORDER ID

В POST `/dock-order` API гарантируется уникальность ORDER ID (аналогично логике CSV импорта).

### Видимость SUPPLIER SLOT

Поле SUPPLIER SLOT скрыто на уровне SLOTBOOK для аккаунтов типа CARRIER и PML — избыточная информация для этих ролей.
