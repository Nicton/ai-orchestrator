# Slot Validation — валидация бронирований

> Сверено с кодом 2026-06-12 | `slotify/whitelist_partners.js`, `slots/resolvers.js:1424`, `site_shipment_statuses/index.js:626`, `zone-slot-validation-settings.js`

## Зачем (бизнес-контекст)

Открыв Slotify, склад получает брони от компаний, которых не знает. Часть из них желанна (новые поставщики клиента), часть — нет (ошибочные брони, ненадёжные перевозчики, занимающие дефицитные окна). Slot Validation — это **фильтр доверия**: проверенные компании бронируют мгновенно, незнакомые попадают в очередь на ручное одобрение. Так склад сохраняет самообслуживание (главную ценность Slotify), не теряя контроль над тем, кто занимает его вместимость.

## Как работает

1. Оператор включает **`is_slot_validation`** на зоне (не на всей площадке — у зон разная критичность: опасные грузы строже, чем обычная приёмка).
2. Оператор ведёт **whitelist** доверенных компаний зоны (`location_zone_whitelist_partners`: zone_id + company_id).
3. При бронировании: компания в whitelist → слот сразу `NEW/APPROVED`; не в whitelist → **`PENDING_VALIDATION`** — окно зарезервировано, но не подтверждено.
4. Оператор разбирает очередь (`buildSlotsPendingValidationQuery` — список ожидающих по адресу): **APPROVED** (бронь подтверждена, бронирующий получает уведомление) или **DECLINED** (окно освобождается).
5. Авто-таймаута нет — решение всегда за человеком (осознанно: автоотклонение чужой брони = потерянный поставщик клиента).

## Кто на что влияет

| Актор | Влияние |
|-------|---------|
| PML-админ | Включает валидацию per-zone; ведёт whitelist (надёжных добавляет — их брони перестают застревать) |
| Оператор с правом `canAccessSlotValidation` | Видит и разбирает очередь; без права PENDING-слоты скрыты из его списков |
| Бронирующий | Видит статус «ожидает подтверждения»; до APPROVED не считает время гарантированным |

## Продуктовые следствия

- PENDING_VALIDATION **резервирует** вместимость — если оператор не разбирает очередь, окна простаивают «в подвешенном» виде. KPI: время реакции на очередь.
- DECLINED — сигнал для бизнеса: повторяющиеся отклонения одной компании = кандидат на разговор с клиентом или в whitelist.
- Whitelist — это и есть «правило заказа слота» для незнакомцев: лучше один раз добавить компанию, чем валидировать каждую бронь.
