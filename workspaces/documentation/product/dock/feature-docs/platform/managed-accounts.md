# Managed Accounts — влияние на систему

> Сверено с кодом 2026-06-12 | `models/account.js:129-136`, `services/accounts.js:80-134`, ACL shared_orders

## Откуда сущность

Холдинг/3PL управляет филиалами и клиентами: филиалу нужен «свой» аккаунт (данные, брендинг, партнёры), но операционно за него работает управляющая команда. Поле **`managed_by_account_id`** (Account → Account) выражает это отношение.

## Что меняется, когда аккаунт managed

| Область | Эффект |
|---------|--------|
| Видимость | Управляющий видит данные managed-аккаунта (заказы, брони); managed видит только своё |
| Действия от имени | Управляющий бронирует слоты/создаёт заявки за managed (роль Booker / shared-доступы) |
| DOCK-заказы | `shared_orders_account_ids` в ACL открывает листинг dock-orders managed-аккаунтов |
| Slotify/Galaxy | Heppner-подобные схемы: carrier-лист и доступы резолвятся через managed-связь Galaxy (`loadManagedShipperAccount`) |
| Ограничения | Создание SR managed-аккаунтом — управляется настройками доступа (не жёсткий запрет в БД); каскад «managed не управляет своим управляющим» |

Продуктовое следствие: «Repeat недоступен для созданного Booker-ом» и подобные правила существуют, чтобы владение операцией оставалось у того, кто её инициировал. При проектировании фич всегда спрашивайте: «как это работает, когда действует управляющий аккаунт?»

## Где найти и как настроить (UI)

**Путь:** **Admin-App** → карточка аккаунта → поле **`managed_by_account_id`** (указать управляющий аккаунт). Настраивается командой Shiptify/админом (не самим клиентом).

После привязки: shared-доступы (`shared_orders_account_ids`) и Galaxy-резолвинг открывают управляющему данные и действия за managed-аккаунт.

## Сценарии

1. **Холдинг + филиалы**: филиалы — managed-аккаунты головного; центральная команда бронирует слоты и ведёт заказы за все филиалы из одного интерфейса.
2. **3PL + клиенты**: 3PL управляет аккаунтами клиентов; при проектировании любой фичи проверяйте «как она ведёт себя, когда действует управляющий аккаунт» (видимость, авторство операции).
