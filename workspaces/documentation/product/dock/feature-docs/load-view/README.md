# Load View — визуализация загрузки склада

> Сверено с кодом 2026-06-12 | `slotify/controllers/planning-slots/load-day.js`, `helper-load-day.js`, `view-load-day.html`

## Зачем (бизнес-контекст)

Календарь слотов отвечает на вопрос «что когда приедет», но не отвечает на вопрос руководителя склада: **«где у меня узкие места и свободные мощности?»**. Load View — это тепловая карта занятости: зоны × часы, интенсивность цвета = % загрузки. По ней принимаются решения: куда направить дополнительную бронь, в какой день предлагать поставщикам окна, где нужно усилить смену, а где персонал простаивает.

## Что показывает

- **Маршрут**: `/slotify/load/:day` (данные — `/api/v1/slots/load-day`, день в таймзоне мастер-локации)
- **Оси**: Y — зоны площадки, X — часы дня; **цвет ячейки** — occupancy: суммарный объём слотов в этот час против вместимости зоны
- **Режимы**: день и неделя (по дням) — оперативный и тактический горизонты
- **Фильтры**: направление (Reception/Expedition), зоны, дата
- **Клик по ячейке** — модал со слотами этого часа: от «здесь красное» до «вот эти три машины» один клик

Расчёт — `builtChartsAndDataZonesBySlots()` (helper-load-day.js): слоты дня группируются по зоне и часу, объём соотносится с capacity зоны.

## Как читать (продуктовые сценарии)

| Паттерн на карте | Что это значит | Действие |
|------------------|----------------|----------|
| Красная вертикаль в 8:00-10:00 | Все хотят утро — пиковая перегрузка | Сдвинуть recurring-слоты, мотивировать дневные окна |
| Пустая зона при загруженных соседних | Зона простаивает (страх валидации? неудобный тип груза?) | Проверить настройки/priority зоны |
| Красный понедельник, пустая пятница | Недельный дисбаланс | Перенастроить pre-advice/окна, договориться с крупными поставщиками |

## Связь с Dock Door Assignment

Та же URL-семья (`/slotify/load/…`) содержит **Assignment View**: сетка время × ворота с drag-and-drop назначением ворот слоту (перенос времени оттуда невозможен — только назначение двери; пересечения подсвечиваются оранжевым, решение за оператором). Load = «сколько», Assignment = «куда именно».

> Примечание (исправлено 2026-06-12): экран **Dock Center существует** — роут `/dock` (4 колонки: Last Slot / Inbound / Outbound / Updates), см. [dock-center](../dock-center/README.md). Load/Assignment/Planning дополняют его аналитикой.
