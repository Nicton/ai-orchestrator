# Slotify — Алгоритм расчёта слотов

Алгоритм определяет длительность и доступность временных окон для конкретного груза в конкретной зоне. Все параметры задаются на уровне зоны (LocationZone).

## Параметры алгоритма

### Min Prior Notice (Минимальное предварительное уведомление)

Минимальное количество дней с текущей даты, с которого разрешено бронировать.

| Значение | Поведение |
|---------|----------|
| `0` | Можно забронировать на сегодня |
| `2` | Ближайшие 2 дня недоступны для бронирования |
| `7` | Можно бронировать только через неделю и позже |

Хранится в `location_zones.settings` → `{ "pre_advice": 7 }`.

### Range (Диапазон)

На сколько дней вперёд открыто бронирование.

| Значение | Поведение |
|---------|----------|
| `7` | Только ближайшие 7 дней |
| `30` | 30 дней вперёд |

Дни за пределами Range отображаются как закрытые в календаре. Хранится в `location_zones.available` → `{ "to": 30 }`.

### Fixed Time Per Slot (Фиксированное время)

Дополнительное время в минутах, добавляемое к каждому слоту сверх расчётного времени груза.

```
Пример:
  Груз: Box 60×30×30 (пропускная способность: 2 ед/час = 30 мин/ед.)
  Заказано: 1 единица
  Базовое время: 30 мин
  Fixed Time Per Slot: 60 мин
  Итоговое время слота: 30 + 60 = 90 мин (1.5 часа)
```

### Slot Interval (Интервал сетки)

Шаг временной сетки для отображения доступных слотов.

| Тип локации | Интервал |
|------------|----------|
| Обычная локация (не ML) | Всегда 15 мин (00:00, 00:15, 00:30 ...) |
| Master Location | Настраивается в зоне (например, 30 или 60 мин) |

Хранится в `location_zones.capacity` → `{ "interval": 30 }`.

### Number of Docks (Количество доков)

Сколько одновременных отправок может быть в один и тот же временной слот.

| Значение | Поведение |
|---------|----------|
| `1` | Один слот за раз — если занят, следующий доступный позже |
| `2` | Два одновременных слота — второй перевозчик тоже может занять то же окно |

```
Пример (Docks = 2):
  Отправка A: 09:00–10:00 (1 дока занята)
  Слот 09:00 ещё ДОСТУПЕН (второй дока свободен)
  → Отправка B тоже может быть на 09:00–10:00
```

## Cargo Capacity (Пропускная способность груза)

В настройках зоны (`location_zones.cargo`) задаётся вместимость для каждого типа груза:

```json
{
  "Box 60x30x30": {
    "performance": 2,
    "unit": "hour"
  }
}
```

### Floor vs Stacked груз

Для каждого типа груза можно задать **отдельные** настройки производительности для груза на полу и стекированного.

```
Пример — заказано 5 ящиков:
  1 → On Floor: производительность 2/час = 30 мин
  4 → Stacked: производительность 4/час = 15 мин/ед.

Расчёт:
  Floor груз: 1 × 30 мин = 30 мин
  Stacked груз: 4 × 15 мин = 60 мин
  Базовое время: 90 мин
  + Fixed Time Per Slot: 60 мин
  Итого: 150 мин → округляется вверх → снаппинг к интервалу (30 мин) = 150 мин
```

## Алгоритм проверки пересечений

`buildTimes(interval, settings)` генерирует список временных корзин на день по недельному расписанию зоны. Каждая корзина имеет флаг `isClosed` (вне рабочих часов).

`getCurrentSlotIndex()` / `getCurrentShipmentIndex()` — маппинг ETA/ETD отправки в конкретную корзину.

```
Отправка A занимает 09:00–10:00 (1 час)
  Попытка на 09:00 → ЗАБЛОКИРОВАНО (перекрытие)
  Попытка на 08:30 → ЗАБЛОКИРОВАНО (начало 08:30 + длит. = 09:30, перекрывает 09:00)
  Попытка на 08:00 → ДОСТУПНО (08:00–09:00, нет перекрытия)
  Попытка на 10:00 → ДОСТУПНО (10:00–11:00, нет перекрытия)
```

Рабочая нагрузка по типам груза вычисляется с учётом стекирования и производительности из `cargo`-конфига через `groupShipmentLoadByCargoType()`.

## Reception vs Expedition

Зоны могут быть настроены для конкретного направления:

| Направление | Поле | Используется для |
|------------|------|------------------|
| Reception | `is_reception=true` | Прибытие в ML (Delivery/Arrival) |
| Expedition | `is_expedition=true` | Убытие из ML (Pickup/Departure) |

В Slotify фильтрация зон:
- `filterSlotsByLocationZone()` — фильтр слотов с учётом направления зоны
- `filterShipmentsByLocationZone()` — фильтр отправок с учётом направления

```
Пользователь выбирает "Deliver TO master location"
→ Показываются только зоны с is_reception=true

Пользователь выбирает "Pickup FROM master location"
→ Показываются только зоны с is_expedition=true
```

## Округление времени

| Правило | Описание |
|---------|----------|
| Дробное время | Всегда округляется **вверх** до целой единицы |
| Slot Interval | Результат "снаппится" к сетке интервала (15, 30, 60 мин) |

Пример: рассчитанные 22 мин при интервале 30 мин → слот занимает 30 мин.
