# Часть 4 — Мини-приложения (Mini-Apps)

> Источник: видео Part 4 (QA-инженер Ярик).

## Обзор мини-приложений

| Мини-приложение | Доступ | Частота |
|----------------|--------|---------|
| Shared Templates (Token Templates) | Токен из настроек Template | Средняя |
| Slotify | Ссылка из настроек зоны ML | **Очень высокая** |
| Slotify Statuses (Airport Screens) | Токен из настроек ML | Высокая |
| Slotify Additional Info | Письмо по email | Высокая |
| Public Tracking Page (PTP) | Письмо по email | Средняя |
| Driver App | SMS на телефон водителя | Средняя |
| Carrier Token Page (CTP) | Токен из настроек Carrier | **Редкая** |
| Jobs (Shiptify Jobs) | Сайт shiptify.com | Низкая |

## Shared Templates (Token Templates)

**Где найти ссылку:** настройки Template → флаг **"Public Share of Templates"** → Generate Token → ссылка.

**Смысл:** поделиться ссылкой с внешним пользователем → он создаёт Booking без доступа к приложению.

### Флаги Template

| Флаг | Значение |
|------|---------|
| **Activate Quote** | Можно создавать Quote Request через этот Template |
| **Activate Direct Booking** | Можно создавать Direct Booking |
| **Modifications Authorized** | Можно изменять локейшены (работает только в обычных Templates, не в Share) |
| **Easy Public** | Template виден **всем** юзерам аккаунта (без — только создателю) |
| **Public Share of Templates** | Включает Share Template функционал |
| **Direct to Carrier** | `On` = Booking сразу уходит Carrier; `Off` = создаётся Draft |

**Процесс:**
1. Создать Template с настройками
2. Сгенерировать токен → получить ссылку
3. Отправить ссылку → пользователь создаёт Booking
4. Booking появляется в приложении (статус зависит от Direct to Carrier)

## Slotify

**Самое активно используемое мини-приложение.**

> "Очень много шипментов создаётся через Slotify. Поэтому даже добавили поля для создания аккаунта — чтобы переманивать людей в полноценное приложение."

**Где найти ссылку:** Настройки Master Location → Настройки зоны → поле URL → ссылка на Slotify.

### Флоу создания через Slotify

```
Ввести email (должен быть без аккаунта!)
    ↓
Выбрать: Deliver TO master location / Pickup FROM
    ↓
Выбрать Зону
    ↓
Выбрать Cargo
    ↓
Выбрать Слот
    ↓
Заполнить Supplier/Customer (вместо второго локейшена)
    ↓
Создать Shipment
```

**Отличие от Slot Booking в приложении:**
- Только **один** локейшн (Master Location)
- Второй локейшн заменяет поле **Supplier/Customer** (отображается на карточке)

**Если email уже существует в системе:**
- Показывается сообщение "У вас уже есть аккаунт — войдите чтобы забукать слот"

## Slotify Additional Info

**Когда использовать:** после создания Shipment через Slotify приходит email с кнопками → одна ведёт на эту страницу.

**Что можно сделать:**
- Заполнить дополнительные поля (метаданные)
- Пример: Driver's First Name → данные добавляются в Shipment
- Нажать **Send Additional Info** → метаданные появляются в Shipment
- Нажать **Cancel** (кнопка залочена намеренно — "чтобы не так хотелось нажимать") → отменить Shipment

**Смысл:** пользователи без доступа к приложению могут добавить информацию.

## Slotify Statuses (Airport Screens)

**Смысл:** экран для отображения на телевизоре на складе.

**Вкладки:**
| Вкладка | Что показывает |
|---------|---------------|
| Upcoming | Shipments, которые ещё не прибыли |
| Delayed | Shipments с опозданием |
| Completed | Shipments с подтверждённым Tracking Point |

**Технические детали:**
- Автообновление каждые **5 минут** (новый запрос к API)
- Показывает только Shipments связанные с **данным Master Location**
- Статус "Completed" = подтверждён соответствующий TP

**Где найти ссылку:** в настройках ML есть несколько сгенерированных ссылок — одна из них на Airport Screen.

## Public Tracking Page (PTP)

**Смысл:** поделиться трекингом Shipment с внешним пользователем (без аккаунта).

**Как создать:**
1. На вкладке Tracking Shipment → блок **Share Shipment Tracking**
2. Ввести email пользователя → нажать Add
3. Пользователь добавляется как **Spectator** (то же, что Spectator через Settings)
4. Ему приходит email со **ссылкой на PTP**

**Что показывает PTP:**
- Название Shipment
- Packing List (список груза)
- Карта с двумя точками (Origin и Destination)
- Даты pickup/delivery
- Carrier

**Дефолтное время 12:00:** если время не выбрано (Full Day) — отображается 12:00.

## Driver App

**Смысл:** приложение для водителей — получить задание и подтвердить Tracking Points.

**Как подключить:**
1. Войти как Carrier
2. На вкладке Tracking Shipment → иконка телефона (внизу, незаметная)
3. Добавить Driver: новый или из существующих
4. Указать **номер телефона** водителя
5. Водителю приходит **реальный SMS** со ссылкой на Driver App

> "Реально летит SMS на реальный телефон."

**Что может водитель в Driver App:**
- Видеть информацию о Shipment
- **Confirm Tracking Points** (прибытие, отправка)
- Добавлять **фото/документы** (POD, CMR и др.)
- Отменять (Cancel) — возможно

Интерфейс Driver App на **французском** языке.

## Carrier Token Page (CTP)

**Очень редко используемое мини-приложение.**

> "Очень редко какие-то тикеты по нему бывают. Я его вообще редко встречаю. Просто для информации, чтобы знать что такое бывает."

**Смысл:** Carrier может создать Booking через публичную страницу (без обычного входа в приложение).

**Где найти:** настройки Carrier аккаунта → блок **"Advanced Customer Booking Templates"** → Generate Token → URL.

**Формат URL:** `[ссылка]/[token]`

**Что делает:** позволяет Carrier создать Booking через форму, которая попадёт в реальный аккаунт.

**Ограничение:** нужно вводить Location вручную (без подсказок/автодополнения).

## Jobs (Shiptify Jobs)

**Смысл:** страница вакансий компании.

**Доступ:** через сайт shiptify.com (раздел вакансий).

**Как работает:**
1. В Admin App → раздел **Jobs** → создать вакансию
2. Вакансия появляется на публичной странице сайта
3. Пользователи могут просматривать и откликаться

Интерфейс на **французском** (т.к. staging Blu убрал английскую версию — только французская).
