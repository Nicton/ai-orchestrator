## Admin App

Административная панель — полный контроль над всеми объектами системы.

> "Со стороны французов только пару человек имеют доступ к реальной админке."

**Доступ:** admin.blu.shiptify.com

### Разделы Admin App

| Раздел | Что содержит |
|--------|-------------|
| **Dashboards** | Статистика по системе |
| **Shippers** | Все аккаунты Shippers — просмотр, редактирование |
| **Carriers** | Все аккаунты Carriers |
| **Accounts** | Все аккаунты (независимо от типа) |
| **Divisions** | Shipper Division / Carrier Division (деления аккаунтов) |
| **Users** | Все пользователи из всех аккаунтов |
| **Admins** | Пользователи с правами администратора |
| **Companies** | Компании (связаны где-то в системе) |
| **Dictionary** | Все справочники системы |
| **Locations** | Все локейшены (даже удалённые пользователями) |
| **Login Events** | Лог входов (кто, когда, что делал) |
| **Integrations** | Интеграции (P44, Shippeo и др.) |
| **Jobs** | Вакансии |
| **Pending Invites** | Отправленные приглашения |

### Dictionary (Словари) в Admin

Здесь создаются **все справочные данные системы**, которые потом выбираются в аккаунтах:

| Словарь | Где используется |
|---------|----------------|
| Shipment Content Types | Типы груза (Cargo Types) |
| Modes | Режимы перевозки (Air, Sea, Road...) |
| Price Details | Типы цен/инвойсов |
| Metadata | Типы метаданных |
| Attachment Types | Типы вложений |
| Message Types | Типы сообщений |
| Currencies | Валюты |

> Принцип: создаётся в Admin → пользователь выбирает из доступного списка в своём аккаунте.

### Управление пользователем в Admin

На странице пользователя:
- Много чекбоксов — каждый за конкретное право
- Поле **"Access to Other Accounts"** — настройка Multi-Account
- Галочка **Multi-Account** + добавление нужных аккаунтов
- Кнопка **Login as User** → войти в любой аккаунт через Admin (без пароля)

### Создание Master Location

Только из Admin можно сделать локейшн Master Location:
> "Сам пользователь не может это сделать — нужно, чтобы в Админке кто-то проставил галочку."

Галочка на уровне локейшна в Admin → "This is a Master Location".

### Доступ к Login

Для входа в приложение нужно:
- Галочка **Login** включена
- НЕТ галочки **Blocked**
- НЕТ галочки **Pending**

Заблокирование происходит автоматически после 5 неправильных попыток входа.

## Back-Office (BO)

**Back-Office** — инструмент для менеджеров (в основном французской команды).

Отличия от Admin:
- Более дружелюбный UI (тоглы, чекбоксы, красивая форма)
- Менее полный функционал
- Для Customer Support, Sales, AM (Account Managers)

**Доступ:** back.blu.shipt.io

### Разделы Back-Office

| Раздел | Описание |
|--------|---------|
| **Accounts** | Настройки аккаунтов в удобном виде |
| **TMS Leads** | Аккаунты с назначенным Owner (AM) |
| **Dashboard Operations** | Статистика по всем shipments, carriers, shippers |
| **Dashboards** | За разные периоды (день, год, 5 лет) |
| **Marketing/Finance** | Специальные страницы для маркетинга |
| **Admin** | Пользователи с фильтром по типу аккаунта |
| **Cargo Groups** | Создание групп Cargo и типов внутри них |
| **Search** | Elastic Search по букингам (по ID, по имени) |

### Login через BO

В BO можно войти в аккаунт:
- Кнопка на аккаунте → войти как первый доступный пользователь
- В разделе Admin → фильтр по Users → Login в конкретного

### Cargo Groups в BO

> "Если создаёшь в Админке — нужно потом на уровне аккаунта добавлять. В BO можно сразу: создать Cargo, засайнить на аккаунт — и оно доступно только для него."

### Статусы пользователей

| Статус | Значение |
|--------|---------|
| **Login** ✅ | Может войти |
| **Blocked** ❌ | Заблокирован (5 неверных паролей) |
| **Pending** ⏳ | Ещё не подтвердил email |

## Galaxy (Галактика)

**История Galaxy:**
> "Galaxy появилась с Heppner — это был самый крупный заказчик, Carrier. Очень крупная компания, у которых было много офисов, складов, структур. Для них придумали Galaxy. Сейчас это стало стандартным функционалом для всех аккаунтов."

**Структура Galaxy:**
```
Galaxy (галактика)
  └── Constellations (созвездия)
        └── Accounts (аккаунты)
```

**Как работает:**
1. Создаётся Galaxy
2. В Galaxy создаются аккаунты (вместо прямого создания аккаунта)
3. Galaxy содержит настройки, которые аккаунты наследуют
4. Shipper Galaxy / Carrier Galaxy

## Terms of Service (Условия использования)

При **первом входе** или после обновления документов:
1. Согласиться с ToS (поставить галочку)
2. Выбрать **Role** (тип роли):
   - Несколько типов ролей
   - В каждом типе — несколько ролей
3. Заполнить Phone (опционально)

Управляется только через **Back-Office** (не Admin).

## FreeTM (Freemium)

Бесплатная версия аккаунта Shiptify.

**Создание через сайт:**
1. Заполнить форму на сайте shiptify.com
2. Получить email → перейти по ссылке
3. Заполнить ещё несколько шагов → создать пароль
4. Создаётся аккаунт с галочкой **Freemium** на уровне аккаунта в Admin

**Отличие от обычного аккаунта:**
- В приложении отображается баннер "You are using free unlimited version of Shiptify"
- Можно конвертировать в обычный аккаунт через Admin

## Public API

**Документация:** apidog.shiptify.com (API Docs)

**Доступные группы роутов:**
- Shipments (получить, создать, обновить)
- Shipment Requests (создать, получить)
- Quote Requests
- Tracking Points (целый блок запросов)
- Metadata (запросить, добавить, список)
- Orders (много роутов, включая Heppner-специфичные)
- Galaxy Scope (дублирование основных роутов)

**API Key:**
- Хранится на уровне пользователя в приложении
- Используется для аутентификации в API запросах
- Через Postman или прямо из apidog.shiptify.com

**Специальный роут (Heppner):**
> "Когда появились Galaxy, для Heppner был специальный запрос создания Shipment Carrier-ом, но от имени Shipper. Heppner был Carrier, но управлял своими Shippers."

## Integrations в Admin

Страница Integrations — список доступных интеграций:
- **P44** (Project44) — автоматические Tracking Points
- **Shippeo** — аналогичная трекинговая платформа
- Другие сторонние сервисы

**Принцип работы:**
- Интеграция привязана к паре **Shipper-Carrier**
- Когда создаётся Shipment между этой парой → интеграция срабатывает
- Могут добавляться дополнительные Tracking Points от стороннего сервиса

Подробнее о технической реализации — см. [06_integrations-technical.md](06_integrations-technical.md).
