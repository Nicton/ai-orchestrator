# Slotify — Алгоритм слотов

> Источник: видео Part 4 (QA-инженер Ярик). Детальное объяснение алгоритма.

---

## Настройки зоны Master Location

Все параметры алгоритма задаются в **настройках зоны** ML:

### Min Prior Notice

**Что означает:** минимальное количество дней до момента, с которого можно создавать Shipment.

| Значение | Поведение |
|---------|----------|
| 0 | Можно букать прямо сегодня |
| 2 | Можно букать только через 2 дня и позже |

### Range (Диапазон)

**Что означает:** на сколько дней вперёд открыто бронирование.

| Значение | Поведение |
|---------|----------|
| 7 | Слоты доступны только в ближайшие 7 дней |
| 30 | Слоты открыты на 30 дней вперёд |

Дни за пределами Range — **закрыты в календаре**.

---

## Fixed Time Per Slot (Фиксированное время слота)

**Что означает:** дополнительное время (в минутах), которое добавляется к каждому слоту.

**Пример расчёта:**
```
Груз: Box 60x30x30 (настройка: можно 2 в час = 30 мин/единица)
Заказано: 1 единица
Базовое время: 30 минут

Fixed Time Per Slot = 60 минут
Итоговое время слота: 30 + 60 = 90 минут (1.5 часа)
```

> "Этот шипмент занял с 6 по 7:30 — полтора часа. Сам ящик занял полчаса, но 60 минут добавились из Fixed Time."

---

## Cargo Capacity (Вместимость груза)

В настройках зоны задаётся **вместимость для каждого типа груза**:

```
Cargo: Box 60x30x30
  → Capacity: 2 в час (= 30 мин на единицу)
  → Stacked настройки (отдельно для стекированного груза)
```

### Floor vs Stacked груз

- **On Floor** — груз на полу (занимает место)
- **Stacked** — груз поставлен на другой груз

**Пример:**
```
Заказано 5 ящиков
1 из них — On Floor (остальные 4 стекированы на него)

Расчёт времени:
  Floor груз → своя настройка времени
  Stacked груз → отдельная настройка (обычно меньше)
```

> "Если 2 ящика, оба On Floor — занимает 1 час. Видно в Planning → Week tab."

---

## Slot Interval (Интервал слотов)

**Что означает:** с каким шагом отображаются доступные слоты.

| Тип локейшна | Интервал |
|-------------|---------|
| Обычный (не ML) | **Всегда 15 минут** (00:00, 00:15, 00:30 ... 23:45) |
| Master Location | Настраивается в зоне (например, 30 минут) |

---

## Number of Docks (Количество доков)

**Что означает:** сколько одновременных Shipment может быть в один и тот же слот.

| Значение | Поведение |
|---------|----------|
| 1 (по умолчанию) | Только 1 Shipment на слот — если занят, слот закрыт |
| 2 | Можно 2 Shipment одновременно (2 дока на складе) |

**Пример:**
```
Docks = 1
Shipment A создан на 9:00-9:30
→ Слот 9:00 занят
→ Следующий доступный: 9:30

Docks = 2
Shipment A создан на 9:00-9:30
→ Слот 9:00 ещё ДОСТУПЕН (второй dok свободен)
→ Можно создать Shipment B тоже на 9:00
```

## Пересечение слотов

Алгоритм учитывает пересечение с высокой точностью:

**Пример:**
```
Shipment A: 9:00-10:00 (занимает 1 час)
Shipment B: попытка на 9:00 → ЗАБЛОКИРОВАНО
Shipment B: попытка на 8:30 → ЗАБЛОКИРОВАНО (пересекается с 9:00)
Shipment B: попытка на 8:00 → ДОСТУПНО (8:00-9:00, не пересекается)
Shipment B: попытка на 10:00 → ДОСТУПНО
```

## Reception vs Expedition

Концепция **направления** для зон:

| Термин | Значение | Аналог |
|--------|---------|--------|
| **Reception** | Прибытие в ML | Delivery/Arrival/Destination |
| **Expedition** | Убытие из ML | Pickup/Departure/Origin |

> "В этом проекте одно и то же называется разными именами: Pickup/Delivery, Origin/Destination, Departure/Arrival, Reception/Expedition. К этому нужно просто привыкнуть."

**Настройка зоны:**
- Зона Reception → только для шипментов, **прибывающих** в ML
- Зона Expedition → только для шипментов, **убывающих** из ML
- Зона оба → доступна для обоих направлений

**Как это работает в CSW:**
```
Создаю Booking:
  Delivery Location = Master Location
  → Показываются только RECEPTION зоны

  Pickup Location = Master Location
  → Показываются только EXPEDITION зоны
```

**В Slotify:**
```
Пользователь выбирает "Deliver TO master location"
→ Показываются только RECEPTION зоны

Пользователь выбирает "Pickup FROM master location"
→ Показываются только EXPEDITION зоны
```

**В Planning:**
```
Toggle RECEPTION → только Shipments с Reception зонами
Toggle EXPEDITION → только Shipments с Expedition зонами
```

## Dock Doors (Двери дока)

Для каждого дока можно создать **несколько дверей (Dock Doors)**:
- В Planning → на Site вкладке отображаются
- Можно назначать конкретную дверь на слот

---

## Zone Calendar (Календарь зоны)

Для каждой зоны есть **собственный календарь** для управления расписанием:

**Что можно сделать:**
- Закрыть конкретный день (праздник, выходной)
- Изменить расписание на один день
- Деактивировать Reception или Expedition на день
- Добавить перерыв
- Сбросить к стандартному расписанию (Reset)

**По умолчанию:** будние дни открыты, выходные закрыты.

---

## ML Status Activations (Статусы)

Для ML можно активировать **кастомные статусы**:
- Статус → Tracking Point (TP)
- Когда статус активируется на Planning странице — соответствующий TP создаётся в Shipment

Настройка в **Settings Master Location** → Status Activations.

## Округление времени

| Параметр | Описание |
|---------|---------|
| Округление | Если расчётное время не целое — округляется до единицы |
| Slot Interval | Слоты привязаны к сетке interval (15, 30, 60 минут) |
