# Часть 1 — Main App: Основной функционал

> Источник: видео Part 1 (QA-инженер Ярик). Staging Blu окружение.

---

## Два типа аккаунтов

В приложении есть две основные роли:
- **Shipper** (грузоотправитель) — создаёт и управляет перевозками
- **Carrier** (перевозчик) — получает и подтверждает заявки

---

## Создание Booking (CSW Wizard)

Нажать кнопку **+BOOK** в сайдбаре:
1. Выбрать **мод** (Air, Sea, Road и др.)
2. Заполнить **Name** (название букинга)
3. Выбрать **тип груза** (Cargo Type)
4. Заполнить **Origin** и **Destination** (откуда/куда)
5. Указать **дату отправки** и **дату прибытия**
6. Перейти на **Step 2** → выбрать **Carrier**
7. Нажать **Send Booking** → создаётся **Booking** (Shipment Request)

После создания попадаем на **детальную страницу букинга** — в URL видим `shipment-request/{id}`.

---

## Жизненный цикл: SR → SH

```
Booking создан
    ↓
Shipment Request (SR) — URL: /shipment-requests/{id}
    ↓
Carrier подтверждает ИЛИ Shipper делает Auto-Confirm
    ↓
Shipment (SH) — URL: /shipments/{id}
    ↑ В URL меняется с shipment-request на shipments
```

**Карточка Booking/Shipment** содержит:
- Имя, перевозчик, адреса, груз
- Кнопка **Auto-Confirm** (со стороны Shipper)
- **Чат** с сообщениями
- Блок **Attachments** (вложения)

---

## Вкладки Shipment

После подтверждения (становится SH) появляются вкладки:

| Вкладка | Что показывает |
|---------|---------------|
| **Booking** | Основная карточка (груз, адреса, даты) |
| **Tracking** | Tracking Points (TP) — статусы отправки/прибытия |
| **Invoicing** | Оплата, прайсы |
| **Claims** | Споры и претензии |

---

## Как видит Carrier

Carrier видит тот же букинг, но:
- Поля связанные с **прайсами** — другие
- Может **подтвердить** или **отклонить**
- Кнопка **Save** для сохранения введённых данных

---

## Статусы Shipment Request

| Статус | Описание |
|--------|---------|
| **Draft** | Введено только имя, сохранён |
| **Ready to Book (RTB)** | Заполнены все поля, ещё не отправлен |
| **On Quote** | Создан QR, ожидание котировок |
| **Booked** | SR создан, ожидает подтверждения |
| **Confirmed** | Подтверждён (стал SH) |
| **Declined** | Отклонён |
| **Closed** | Закрыт |

**Draft** создаётся нажатием SAVE на шаге 1 CSW с минимальными данными (только имя).
**Ready to Book** — заполнены все поля + выбран carrier на шаге 2, нажат SAVE (не отправлен).

---

## Quote Request (QR)

Отличие от Direct Booking:
- В CSW нажать **Request a Quote** вместо Direct Booking
- Можно выбрать **несколько carriers** одновременно
- Каждый Carrier получает запрос и может прислать своё предложение с ценой
- Shipper выбирает лучшее предложение → создаётся Shipment с выбранным Carrier

**Со стороны Carrier при QR:** нет кнопки Confirm — только заполнить прайс и отправить котировку.

---

## Tracking, Invoicing, Claims

**Tracking** (вкладка после confirm):
- URL меняется на `/shipments/{id}`
- Tracking Points (TP): отправление + прибытие
- Можно **Replan** (перепланировать) или **Request Info**
- Кнопка **Open Claim** → появляется вкладка Claims

**Invoicing:**
- Типы прайсов, добавление цен
- Assign на месяц, ввод суммы и номера инвойса

**Claims** (Disputes):
> "Есть байка, что приложение изначально задумывалось именно как инструмент для клеймов. Со временем выросло в большую платформу, но клеймы остались."

## Multi-Container (MC)

Когда выбирается тип груза **контейнер** (вкладка FTL в CSW):
- Это автоматически называется **Multi-container** (даже если 1 контейнер)
- При Confirm: **1 контейнер = 1 Shipment**
- Один SR/Booking → несколько SH (по количеству контейнеров)
- В Tracking List отображаются отдельными строками

---

## Элементы карточки Booking/SH

**Что можно обновить:**
- Имя
- Локейшены (адреса) и даты/время
- Груз (тип, количество)
- Документы (CMR и др.)

**Attachments (вложения):**
- Выбрать тип атачмента → загрузить файл
- Появляются в блоке атачментов

**Metadata (MD):**
- Можно **поделиться** (share) или **запросить** информацию
- Пример: "Driver FirstName = Ярик"
- Видно в блоке атачментов с типом и значением

---

## Templates (Шаблоны)

Страница Templates — сохранённые шаблоны букингов:
1. Создать Template → заполнить поля
2. Нажать **Book** из шаблона → открывается CSW с предзаполненными данными
3. Дозаполнить имя и даты → создать букинг

Также из сайдбара: +BOOK → Templates → выбрать шаблон.

---

## Routines (Рутины)

Рутина = **группа шаблонов**:
- Одна рутина содержит несколько Template строк
- **Типы:** Daily (на конкретный день) и Weekly (на конкретную неделю)
- Запуск: из сайдбара выбрать рутину → выбрать дату/неделю → все шаблоны создаются как готовые букинги одним нажатием

---

## Multi-Booking и Multi-Confirm

**Multi-Booking (Ready to Book):**
- Кнопка "Сколько дат записи" (Multi-Book)
- Все букинги в статусе RTB → одна кнопка → создаются сразу

**Multi-Confirm:**
- Выбрать статус "Booked" → появляется чекбокс для всех
- Кнопка "Confirm N букингов" → все одновременно конфирмируются (становятся SH)

---

## Cargo Types и Settings

Страница Cargo Types:
- Слева: **все доступные** в системе типы груза
- Справа: **выбранные** для моего аккаунта
- Активируются для конкретных модов (Air, Sea, Road)

Аналогично работают: Metadata Types, Attachment Types и другие страницы настроек.
